# 算法工程师面试重点（数据结构 + 算法 + ML/DL + 工程落地）

## 复杂度分析
- 时间复杂度：O(1)/O(log n)/O(n)/O(n log n)/O(n²)/O(2ⁿ)，主定理求解递推式。
- 空间复杂度：辅助空间 vs 递归栈空间，原地算法（in-place）优化。
- 均摊分析：动态数组扩容、并查集路径压缩的均摊复杂度。
- 最好/最坏/平均复杂度对比，以及实际工程中关注 P95/P99 的原因。

## 数组与链表
- 数组：连续内存、随机访问 O(1)、插入删除 O(n)，动态数组扩容策略（2倍/1.5倍）。
- 链表：单链表/双向链表/循环链表，插入删除 O(1)、查找 O(n)。
- 高频思路：双指针（快慢指针/对撞指针）、滑动窗口、前缀和、差分数组。
- 原地算法：荷兰国旗问题、数组去重、移动零、旋转数组。

## 栈与队列
- 栈：后进先出，单调栈（下一个更大/更小元素）、表达式求值、括号匹配。
- 队列：先进先出，双端队列（Deque）、循环队列、优先队列（堆实现）。
- 栈实现队列 / 队列实现栈：双栈/双队列互换，摊还分析。

## 哈希表
- 原理：哈希函数 + 冲突解决（链地址法/开放寻址法/再哈希）。
- 负载因子与扩容：rehash 过程、渐进式 rehash（Redis dict）。
- 应用：去重、频次统计、两数之和、LRU Cache（哈希+双向链表）。
- 一致性哈希：虚拟节点解决数据倾斜，新增/移除节点时最小化数据迁移。

## 树
- 二叉树：前中后序遍历（递归/迭代/Morris）、层序遍历、序列化与反序列化。
- 二叉搜索树：查找/插入/删除 O(h)，平衡的重要性。
- 平衡树：AVL（严格平衡，旋转操作）、红黑树（近似平衡，颜色约束）。
- B/B+ 树：多路平衡搜索树，B+ 树叶子节点链表，适合磁盘索引。
- 堆：完全二叉树，数组实现，建堆 O(n)，TopK、动态中位数、多路归并。
- Trie 树：前缀树，自动补全、单词搜索、IP 路由最长前缀匹配。
- 并查集：路径压缩 + 按秩合并，连通分量、最小生成树（Kruskal）。

## 图
- 存储：邻接矩阵 vs 邻接表，稠密图 vs 稀疏图选型。
- 遍历：BFS（队列，最短路径无权重）、DFS（栈/递归，连通性）。
- 最短路径：Dijkstra（非负权）、Bellman-Ford（负权）、Floyd（全源）。
- 最小生成树：Prim（点贪心）、Kruskal（边贪心+并查集）。
- 拓扑排序：Kahn 算法（入度表）/ DFS 后序逆序，检测有向环。
- 强连通分量：Tarjan/Kosaraju 算法。

## 排序算法
- 比较排序：快排（分治、原地、平均 O(n log n)）、归并（稳定、O(n log n)）、堆排。
- 非比较排序：计数排序、基数排序、桶排序，适用整数/固定范围场景。
- 排序稳定性：归并/插入/冒泡稳定，快排/堆排/选择不稳定。
- 工程排序：Java `Arrays.sort()`（Dual-Pivot QuickSort + Timsort）、C++ `std::sort`（IntroSort）。

## 二分搜索与变种
- 标准二分：闭区间/开区间/左闭右开写法，终止条件与边界处理。
- 变种：查找第一个/最后一个满足条件的元素、旋转排序数组查找、平方根。
- 二分答案：最大值最小化/最小值最大化，单调性判定。

## 动态规划
- 核心要素：状态定义、状态转移方程、初始化、遍历顺序、边界条件。
- 常见类型：线性 DP（LIS/LCS）、背包（0-1/完全/多重/分组）、区间 DP、树形 DP、状态压缩 DP。
- 空间优化：滚动数组（只用前两行/前一列）、一维数组倒序遍历（0-1 背包）。
- 常见失误：状态遗漏、重复计算、初始化错误、覆盖顺序错误。
- 与贪心的区别：DP 保留所有子问题解，贪心只保留局部最优。

## 贪心算法
- 核心：局部最优能否推导全局最优，需给交换论证或反例说明。
- 经典问题：活动选择、区间调度、霍夫曼编码、最小生成树。
- 与 DP 的取舍：具有最优子结构+贪心选择性质时可用贪心，否则用 DP。

## 回溯与搜索
- 回溯：DFS + 剪枝，搜索树、状态恢复（撤销选择）。
- 剪枝策略：可行性剪枝、最优性剪枝、对称性剪枝。
- 排列/组合/子集生成，去重策略（排序+跳过重复元素）。
- 启发式搜索：A* 算法（f=g+h），八数码、最短路径问题。

## 字符串算法
- KMP：前缀函数（最长相等前后缀）、模式匹配 O(m+n)。
- Trie 树：前缀匹配、自动补全。
- 后缀数组/后缀自动机：最长重复子串、不同子串个数。
- 马拉车算法（Manacher）：最长回文子串 O(n)。
- Rabin-Karp：滚动哈希，多模式匹配。

## 机器学习基础
- 监督学习 vs 无监督学习 vs 强化学习，各自适用场景。
- 线性回归 vs 逻辑回归：最小二乘法 vs 极大似然估计，正则化（L1/L2）。
- 决策树：ID3（信息增益）、C4.5（信息增益率）、CART（Gini 指数）。
- 集成学习：Bagging（随机森林，降低方差）、Boosting（GBDT/XGBoost/LightGBM，降低偏差）。
- SVM：最大间隔分类器，核技巧（Kernel Trick）处理非线性，支持向量。
- 模型评估：准确率/精确率/召回率/F1/AUC-ROC/PR 曲线，过拟合与欠拟合。
- 交叉验证：K-Fold、留一法，模型选择 vs 超参数调优。

## 深度学习基础
- 神经网络：前向传播、反向传播（链式法则）、激活函数（ReLU/Sigmoid/Tanh/Swish）。
- 优化器：SGD/Momentum/AdaGrad/Adam/AdamW，学习率调度策略。
- 正则化：Dropout、Batch Normalization、Layer Normalization、数据增强。
- CNN：卷积、池化、感受野、经典架构（LeNet/AlexNet/VGG/ResNet/EfficientNet）。
- RNN/LSTM/GRU：序列建模、梯度消失、门控机制。
- Transformer：自注意力（Self-Attention）、多头注意力、位置编码、Encoder-Decoder。
- BERT/GPT：预训练（MLM/CLM）+ 微调范式，迁移学习。

## 深度学习框架
- PyTorch：动态计算图、自动微分（autograd）、nn.Module、DataLoader、分布式训练（DDP）。
- TensorFlow：静态计算图（1.x）/ Eager Execution（2.x）、SavedModel、TF Serving。
- 模型保存与加载：state_dict、Checkpoint、ONNX 跨框架导出。
- 混合精度训练：FP16/BF16 + 损失缩放，减少显存占用与加速训练。

## 工程落地
- 模型部署：TorchScript/ONNX/TensorRT/TVM，量化（INT8/FP16）压缩模型。
- 推理优化：批处理（Batching）、动态批处理（Dynamic Batching）、模型并行/流水线并行。
- 特征工程：特征存储（Feature Store）、离线/在线一致性、特征漂移检测。
- A/B 测试：分流策略、样本量计算、显著性检验（t-test/卡方检验）、指标选择。
- 在线学习：增量训练、模型热更新、冷启动问题。
- 效果归因：因果推断（PSM/DID）、 uplift 建模、多触点归因。

## 系统设计（算法方向）
- 推荐系统：协同过滤（User/Item CF）、矩阵分解、双塔模型、召回→粗排→精排→重排。
- 搜索排序：Learning to Rank（Pointwise/Pairwise/Listwise）、BM25、向量召回。
- 广告系统：CTR/CVR 预估、竞价机制（GSP/VCG）、预算控制、频率控制。
- 风控系统：规则引擎 + 模型融合、实时特征、黑白名单、策略迭代。

- 这个算法的时间/空间复杂度是多少？能优化吗？

- 动态规划的状态定义是怎么想到的？有没有其他定义方式？
- 模型在线上效果下降了，你怎么排查？是数据漂移还是模型老化？
- A/B 测试结果不显著，可能是什么原因？样本量够吗？
- 模型推理延迟要求 <10ms，你会怎么做优化？
- 特征工程在你的项目中做了什么？哪些特征增益最大？
