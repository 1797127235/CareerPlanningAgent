# Design Context · 职途智析

最后更新：2026-04-15

> 本文档定义 design 决策的根。所有 design skill（impeccable 系列）都从这里取上下文。
> 改动这份文档前，请确认与 owner 已对齐。

---

## Users

中国 CS/IT 大三/大四在校学生（19-23 岁），处于职业方向迷茫期或求职紧迫期。

**使用场景**：PC web 为主，常常在情绪不稳定的时刻打开 — 晚自习焦虑、看到同学拿 offer 后、面试挂掉之后、被父母催了之后。

**Job to be done**：他们要的不是"工具"，是**懂他们处境的伙伴**。在迷茫时被看见，在焦虑时被托住，在做完一件小事时被认可。

---

## Brand Personality

**温暖 · 同行 · 不评判**

- **温暖**：不是 warm-and-fuzzy 鸡汤温暖，是"懂你正在经历什么"的具体温暖。引用用户具体的事，不说"加油"
- **同行**：像懂你的学长学姐（早你 1-2 届的人）。**不是导师/老师/HR/AI 助手**
- **不评判**：用户做了什么或没做什么，系统先看见再分析，绝不"你应该 / 你必须"

**情绪目标**：让用户感到 **不孤单 + 不被审判 + 被听见**。

## Aesthetic Direction

**晨光编辑体**（Editorial Morning Light）

抛弃当前的 liquid glass（清新科技感 — 但太"产品感"），转向：

### 视觉语言
- **纸感底色**：奶米/淡象牙色，配合极细颗粒纹理（noise overlay 1-2% opacity）
- **章节感排版**：long form editorial 风格，大量留白，章节化分节（用罗马数字 / "CHAPTER N · ..." 标签开头）
- **温润色板**：苔绿 + 橙土 + 深栗 — 自然、有温度，不是 AI 默认的 cyan/purple
- **混排字体**：衬线 display + 几何 sans + 中文柔和体

### 色板（OKLCH，必用）
```css
--bg-paper:    oklch(0.97 0.012 75);   /* 奶米底 - 主背景 */
--bg-card:     oklch(0.99 0.006 75);   /* 卡片 / 区块 */
--ink-1:       oklch(0.20 0.018 60);   /* 深栗墨 - 主文字 */
--ink-2:       oklch(0.42 0.012 60);   /* 段落次级 */
--ink-3:       oklch(0.62 0.008 60);   /* 辅助 / 时间戳 */
--moss:        oklch(0.55 0.090 145);  /* 苔绿 - 主 accent / 进度 */
--ember:       oklch(0.66 0.13 50);    /* 橙土 - 行动召唤 / 提醒 */
--chestnut:    oklch(0.32 0.05 30);    /* 深栗 - 章节标题 / quote */
--line:        oklch(0.88 0.012 75);   /* 细线 - 分隔 */
```

**禁用色**：纯黑 #000、纯白 #fff、cyan #00bcd4、紫色 gradient（紫→蓝是 AI slop top 1）。

### 字体（避开 impeccable 禁用列表）
- **Display**（章节标题 / hero）：**Bricolage Grotesque**（Google Fonts，variable，character 强但不学究）
- **Serif**（quote / chapter label）：**Source Serif 4**（Google Fonts，温度足）
- **Body**（正文 / UI）：**Geist Variable**（已有，保留）
- **中文**：**Noto Sans SC**（已有，保留） + **Noto Serif SC**（章节标题中文）

**禁用字体**：Plus Jakarta Sans / Inter / DM Sans / Roboto / Open Sans / 系统默认。

### 排版规则
- 中文 line-height：base + 0.1（中文字符比拉丁系字符视觉更"实"，需要更多呼吸）
- 段落最大宽度：65-75ch
- 字号比例：1.25 ratio 起步，hero 区跳到 3-5x（不是平庸的 1.5x）
- Page padding：移动 24px / 桌面 64-96px（**不要顶满**，留白即温度）

### 视觉效果
- **不要**：glassmorphism、cyan/purple gradient、border-left 色条、gradient text、large icon-above-heading 模板感
- **要**：极细颗粒纹理（noise）、温暖的 shadow（不是 cool blue 的 drop shadow）、章节分隔线、罗马数字 hero label

### 主题
**浅色（默认且唯一）** — 温暖与浅色协同。深色模式留作未来扩展，本期不做。

## Design Principles（编码 design 决策时回溯这 5 条）

1. **温度优先于精确** — 数字诚实给（覆盖率 38%），但语言软化（"已经走完三分之一"，不是"完成度 38%"）
2. **共情前置于分析** — 用户每个状态先被命名再被分析（"看到同学拿 offer 你比谁都急 — 这感受是真的"）
3. **零评判用语** — 删 "应该 / 必须 / 建议你要 / 你需要"；替换为 "可以试试 / 哪个先动 / 想要哪种"
4. **章节式叙事** — 页面像翻书的章节，不是看仪表盘。每区块有 标签 + 标题 + 段落 + 行动
5. **留白即温度** — 拒绝把屏幕填满。留白是给用户消化情绪的空间

## 反参考（Anti-references）

明确不要做成的样子：

| 反参考 | 不要的具体特征 |
|---|---|
| **教育产品（小猿 / 网易云课堂）** | 强营销 banner / 推销倒计时 / 花哨广告位 / 9.9 元限时课卡片 |
| **Boss 直聘 / 智联** | 商业感强 / 信息密度爆炸 / PUA 用户感 / 红色徽标到处 |
| **AI slop 产物** | cyan-purple gradient / glassmorphism / 大圆角 + drop shadow 卡片矩阵 / hero metric 模板 |

## Constraints

- WCAG AA（必达）— 文字对比度、focus ring、reduced-motion
- 中文主语言：line-height +0.1，避免过窄字体
- 技术栈：Tailwind v4 + shadcn/ui + Radix Primitives 已就位
- **现有数据 schema 不动** — 视觉重写不能破坏 backend API 契约或 React Query key
- 不引入新 npm 依赖（字体走 Google Fonts / @fontsource）

---

## Differentiation

**用户离开时记住的不是"用了一个 AI 工具"，而是"被一个懂我的学长聊了几句"。**

如果某个 design 决策让产品看起来更像"工具"或"教育平台"或"商业 SaaS"，回到本文档对照。
