## Design Context

### Users

Young Chinese job-seekers and early-career planners (approximately 18-30). They
open this product **anxious, sometimes late at night, sometimes on the way to an
interview**. They want to see, at a glance:

- Where they stand right now (profile completeness, zone, top gaps)
- What changed since last visit (new reports, new analyses)
- What to do next (one clear action, or a short list of candidates)

They do NOT want to be greeted by a magazine cover.

### Brand Personality

Three concrete words (not "modern/clean/elegant" — those are dead categories):

**柔软 · 可靠 · 像一本自己的档案本**

Soft-feeling paper (not glossy). Reliable like a reference book you trust. Like
a leather notebook you write in — your own, not a public magazine. Every page is
useful before it is pretty.

### Aesthetic Direction

**Reference physical object**: A Muji-style hardcover notebook, cream paper,
unruled, with pencil divisions between sections. A personal file — not
editorial, not Linear-cold, not brutalist. Information-dense but warm. Tools
first, text second. Close to how Notion and Craft feel, but with more
temperature.

**Anti-references**
- Wired/New Yorker magazine heroes (that was v2)
- Linear / Vercel dark dashboards (too cold for anxious users)
- AI-typical purple-cyan gradients, glass cards, neon accents
- SaaS slate-and-blue (v1 original)

**Theme**: **Light** — cream-warm paper base. Users often open late at night,
but the right answer for anxiety + reading-data is calm warm light, not dark
mode (which adds weight and makes small dense text harder to read).

### Design Principles

1. **第一屏必须承诺价值** — every non-reading page must surface the top 3 data
   points or the top action in the first viewport (no scrolling required).
   Reading-type pages (Report, Coach narrative/review) are the ONLY exception.
2. **Block over scroll** — dense, independent blocks separated by hairlines or
   whitespace. No long vertical storytelling. Page = notebook page, not
   article.
3. **Accent is rare** — chestnut (栗色) only for emphasized numbers, active
   states, primary CTAs. Overuse makes it meaningless.
4. **Serif is seasoning** — Literata italic only for quotes, short emphasized
   callouts, and date/timestamp captions. It is NOT the body font.
5. **Density with air** — gap-4 between blocks, not py-24 between sections.
   Compact but breathable. Reader scans like a real notebook, not forced down
   a single column.

## Design Tokens

### Typography

Chosen through font_selection_procedure (rejected Inter / DM Sans / Plus Jakarta
Sans / Fraunces / Source Serif / Newsreader — all in the banned reflex list).

- **Primary sans**: **Commissioner** (Google Fonts, variable, 100-900 weights)
  — humanist geometric, slightly warmer than Inter, built for reading UI at
  small sizes. Not Geist, not Söhne, not Inter.
- **Accent serif**: **Literata** (Google Fonts, variable, Play Books reading
  serif) — for italic emphasis, quotes, and date captions only.
- **Mono**: **JetBrains Mono** (retained from v2) — for numbers, code, IDs.
- **Chinese**: `Noto Sans SC` (body), `Noto Serif SC` (accent italic).

```css
--font-sans: "Commissioner Variable", "Noto Sans SC", system-ui, sans-serif;
--font-serif: "Literata Variable", "Noto Serif SC", Georgia, serif;
--font-mono: "JetBrains Mono", ui-monospace, monospace;
```

### Type scale (fixed rem, no clamp display-xl)

App UI, not marketing. Fixed scale, clear 1.25+ contrast between steps.

| token | px | role |
|---|---|---|
| `--text-2xs` | 10 | meta caption |
| `--text-xs` | 11 | kicker / badge |
| `--text-sm` | 13 | block body sub |
| `--text-base` | 15 | block body |
| `--text-lg` | 17 | block title |
| `--text-xl` | 21 | section heading |
| `--text-2xl` | 28 | page title |
| `--text-3xl` | 40 | display (rare — page-level identity only) |

**Max display**: 40px. v2's 76px is retired for non-reading pages.

### Colors (OKLCH)

Paper is cream, not white. Ink is warm dark, not black. Accent is chestnut with
disciplined neighbors (moss, ember).

```css
--bg-paper:       oklch(0.98  0.008 80);
--bg-paper-2:     oklch(0.965 0.010 80);
--bg-card:        oklch(0.995 0.005 80);
--bg-card-hover:  oklch(0.985 0.008 80);

--ink-1:          oklch(0.22  0.010 50);
--ink-2:          oklch(0.48  0.010 50);
--ink-3:          oklch(0.72  0.008 50);

--line:           oklch(0.90  0.010 80);
--highlight:      oklch(0.94  0.025 85);

--chestnut:       oklch(0.42  0.10  30);
--chestnut-soft:  oklch(0.92  0.04  30);
--moss:           oklch(0.58  0.09  145);
--ember:          oklch(0.65  0.13  40);
--ember-soft:     oklch(0.94  0.04  40);
```

Neutral hue is tinted 50-80° (warm yellow-red) so the whole palette feels
continuous with the chestnut accent. No pure white, no pure black.

### Spacing (4pt scale, semantic names)

```css
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;   /* default gap between blocks */
--space-5: 24px;   /* default between block groups */
--space-6: 32px;
--space-7: 48px;
--space-8: 64px;
--space-9: 96px;   /* page-level margins only */
```

Default vertical rhythm: between blocks `gap-4 / gap-5`, between groups
`gap-6 / gap-7`. **Never `py-16` / `py-24` between adjacent blocks** (that was
v2).

### Elevation

```css
--shadow-block: 0 1px 2px oklch(0.50 0.02 50 / 0.06);
--shadow-float: 0 4px 12px oklch(0.50 0.02 50 / 0.10);
```

Minimal — the hairline border does most of the edge work.

### Radius

```css
--radius-sm: 4px;
--radius-md: 6px;     /* default for blocks */
--radius-lg: 10px;    /* for larger cards / modals */
--radius-pill: 9999px; /* tags / avatars */
```

Notebook-like small radii. Not the `rounded-2xl` AI-slop default.

## Block Patterns

The building block is a **block**, not a card-with-shadow:

```
┌─ Block ──────────────────────────┐
│ KICKER (uppercase 11px)          │
│ Block title (text-lg semibold)   │
│ ───── hairline ─────              │
│ content rows, text-sm / base     │
│ row 2, row 3...                  │
│                                  │
│ [optional footer row]            │
└──────────────────────────────────┘
```

Multiple blocks stack with `gap-4`, and blocks often go side-by-side in
`grid-template-columns: repeat(auto-fit, minmax(280px, 1fr))`. The layout is
**asymmetric** — not all blocks the same size.

**Callout blocks** use `--chestnut-soft` or `--ember-soft` background fill (no
side-stripe borders — those are banned).

**Timeline / list blocks** use hairline dividers between rows, no nested cards.

**Empty states** teach the interface: "还没有任何 X — 去 [action] 记一笔" +
link, not "Nothing here yet."

## Motion

Minimal. A notebook doesn't animate.

- **Entry**: fade in 150ms on route change. No staggered reveals.
- **Interactive feedback**: opacity 200ms on hover, scale 0.98 on press.
- **State transitions**: grid-template-rows for expanding blocks.
- **Easing**: `cubic-bezier(0.22, 1, 0.36, 1)` (expo out) everywhere.
- **Reduced motion**: all transitions → 0.01ms.

## Per-Page Layout Rules

| page | role | first screen must show |
|---|---|---|
| HomePage | dashboard | greeting line + 3 navigation blocks (Profile / Report / Growth) |
| ProfilePage | archive | name + 6-section progress + 3 primary blocks (education / skills / target) |
| GraphPage | explorer | 4-zone compact map (all zones visible at once, not chapters) |
| CoachResultPage (JD diagnosis) | **decision surface** | readiness % + matched count + gap count + primary CTA |
| CoachResultPage (narrative / review) | **reading** — exception | may keep bigger hero + markdown body |
| CoachResultsListPage | list | list rows with date / type / title / quick delete |
| CoachChatPage | conversation | small kicker + input + messages (no big hero) |
| LoginPage | input | tab + form (no hero over 32px) |
| ReportPage | **reading** — exception | may keep editorial layout from v2 |

**Reading exception** applies only when the core user task is to read long text
end-to-end. Everything else is a tool page.

## Components to retain / retire from v2

**Retain**
- `PaperCard` (with onClick + keyboard a11y from e7de8ad) — re-skin to notebook
- `Kicker` — still used (11px uppercase)
- `TableOfContents` — reading pages only

**Retire / replace**
- `Chapter`, `ChapterOpener` — big-hero + staggered reveal is reading-page only
- `DropCap`, `PullQuote` — reading-page only
- `--fs-display-xl / --fs-display-lg / --fs-display-md` → replaced by `--text-*`

**New primitives to add**
- `Block` — notebook page unit (kicker + title + hairline + rows)
- `BlockGrid` — auto-fit asymmetric grid container
- `DataRow` — key / value row inside block
- `InlineTag` — small chip for skills / zones
- `Callout` — soft-background block for emphasis (no side stripes)

## AI Slop Self-Test (v3)

If someone sees this interface and says:
- "Oh, it looks like Notion" → bad (not differentiated)
- "Oh, it looks like Linear" → bad (wrong audience)
- "Oh, AI made this" → worst
- "This looks like a personal notebook / file" → right
- "It's dense but calming" → right
- "Every screen I can act on without scrolling" → right

The identity lives in:
1. Cream paper + chestnut accent (not gray + blue)
2. Commissioner (not Inter) + Literata italic (not Fraunces)
3. Compact blocks (not big cards / big heroes)
4. Hairline borders (not heavy shadows)
5. Serif only as seasoning (not body)
