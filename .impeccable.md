最后更新：2026-04-15 · 晨光编辑体 v2.0

> 本文档定义 design 决策的根。所有 design skill（impeccable 系列）都从这里取上下文。
> **v2.0 变更**：在 v1 晨光编辑体基础上扩展到一流 editorial 水准（New Yorker / Medium long read 级），适用对象从"GrowthLog 单模块"扩展到"**v2 全前端**"（frontend-v2/ 子项目）。

## Users

中国 CS/IT 大三/大四在校学生（19-23 岁），处于职业方向迷茫期或求职紧迫期。

**使用场景**：PC web 为主，常常在情绪不稳定的时刻打开 — 晚自习焦虑、看到同学拿 offer 后、面试挂掉之后、被父母催了之后。

**Job to be done**：他们要的不是"工具"，是**懂他们处境的伙伴**。在迷茫时被看见，在焦虑时被托住，在做完一件小事时被认可。

---

## Brand Personality

**温暖 · 同行 · 不评判**

- **温暖**：不是 warm-and-fuzzy 鸡汤温暖，是"懂你正在经历什么"的具体温暖。引用用户具体的事，不说"加油"
- **同行**：像懂你的学长学姐（早你 1-2 届的人）。**不是导师/老师/HR/AI 助手**
- **不评判**：用户做了什么或没做什么，系统先看见再分析，绝不"你应该 / 你必须"

**情绪目标**：让用户感到 **不孤单 + 不被审判 + 被听见**。

## Aesthetic Direction

**晨光编辑体 v2.0**（Editorial Morning Light — 一流 editorial 水准）

继承 v1 的奶米/苔绿/橙土/深栗 + 衬线 display + 章节制。v2.0 新增三维度升级：

### 视觉语言（v2.0 强化）

- **纸感底色**：奶米/淡象牙色 + **落地 noise/grain overlay**（v1 说要做但没做到）
- **章节感排版**：long form editorial 风格，**Hero 区极致大字**（clamp 到 72px+），大量留白
- **温润色板**：苔绿 + 橙土 + 深栗 + **warm shadow**（替代默认 cool blue drop shadow）
- **混排字体**：衬线 display + 几何 sans + 中文柔和体
- **Editorial 装饰**：DropCap 首字母放大 / PullQuote 引言 / Kicker 导语 / ChapterOpener 章节开篇

### 色板（OKLCH，v2.0 扩展）

```css
/* v1 继承（保持不变） */
--bg-paper:    oklch(0.97 0.012 75);   /* 奶米底 */
--bg-card:     oklch(0.99 0.006 75);   /* 卡片 / 区块 */
--ink-1:       oklch(0.20 0.018 60);   /* 深栗墨 - 主文字 */
--ink-2:       oklch(0.42 0.012 60);   /* 段落次级 */
--ink-3:       oklch(0.62 0.008 60);   /* 辅助 / 时间戳 */
--moss:        oklch(0.55 0.090 145);  /* 苔绿 - 主 accent */
--ember:       oklch(0.66 0.13 50);    /* 橙土 - 行动召唤 */
--chestnut:    oklch(0.32 0.05 30);    /* 深栗 - 章节标题 */
--line:        oklch(0.88 0.012 75);   /* 细线 - 分隔 */

/* v2.0 新增 */
--bg-paper-2:     oklch(0.955 0.014 75);    /* 次级底色（章节间背景切换） */
--bg-card-hover:  oklch(0.985 0.008 75);    /* 卡片 hover 态 */
--highlight:      oklch(0.93 0.045 85);     /* 淡金 - 引文/highlight 背景 */
--warm-shadow-1:  oklch(0.65 0.025 55 / 0.08);  /* 暖阴影 L1（替代 cool shadow）*/
--warm-shadow-2:  oklch(0.50 0.035 50 / 0.12);  /* 暖阴影 L2 */
--grain-color:    oklch(0.25 0.02 60 / 0.025); /* 颗粒纹理颜色 */
```

**禁用色**：纯黑 #000、纯白 #fff、cyan #00bcd4、紫色 gradient（紫→蓝是 AI slop top 1）。

### 字体（v1 继承 + v2.0 扩展）

- **Display**（章节标题 / hero）：**Bricolage Grotesque**（Google Fonts, variable）
- **Serif**（quote / chapter label / DropCap）：**Source Serif 4**（Google Fonts，温度足）
- **Body**（正文 / UI）：**Geist Variable**
- **中文**：**Noto Sans SC**（body） + **Noto Serif SC**（章节标题 / pullquote 中文）
- **v2.0 新增** — **Mono**（极少用，仅代码片段 / 日期戳 tabular）：**JetBrains Mono**

**禁用字体**：Plus Jakarta Sans / Inter / DM Sans / Roboto / Open Sans / 系统默认。

### Typography Scale（v2.0 — 极致化）

v1 flat hierarchy 的问题解决：

```css
/* Display / Hero — 页面顶部「这是一章开始」的大字 */
--fs-display-xl:   clamp(42px, 6vw, 96px);   /* 章节 opener / 封面级 */
--fs-display-lg:   clamp(36px, 5vw, 72px);   /* 每章 hero */
--fs-display-md:   clamp(28px, 3.6vw, 52px); /* 次级章节 */

/* Reading body — 长 form 阅读区 */
--fs-body-lg:      19px;   /* 主段落（桌面），line-height 1.75 */
--fs-body:         16px;   /* 次级正文 */
--fs-body-sm:      13px;   /* 辅助 */
--fs-caption:      11px;   /* 时间戳 / 标签 */

/* Pull quote / emphasis */
--fs-quote:        clamp(24px, 2.8vw, 40px); /* 引言斜体 */

/* Line heights */
--lh-display:      1.12;   /* 紧凑，display 字号不需要更多行距 */
--lh-body:         1.75;   /* 长 form reading 舒适行距 */
--lh-body-zh:      1.85;   /* 中文 +0.1 */
--lh-ui:           1.4;    /* UI 控件 */
```

**Ratio**：display 4 级之间 ≥ 1.4 ratio；body 4 级之间 1.3 ratio（比 v1 的 flat 1.1 大幅提升）。

**字重**：
- Display：**weight 500**（medium — 不是 700 bold，保留编辑感优雅）
- Body：**400** normal，**semibold 600** emphasis；**never bold 700**（长 form 不要重字）
- 中文 display：**weight 400-500**（中文 bold 视觉太重）

### Paper Texture（v2.0 — 落地 v1 未做到的 noise）

**全局 noise overlay**：

```css
/* body 全局纹理 */
body::before {
  content: "";
  position: fixed;
  inset: 0;
  pointer-events: none;
  z-index: 1;
  background-image: url("data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 200 200'><filter id='n'><feTurbulence type='fractalNoise' baseFrequency='0.85' numOctaves='2' stitchTiles='stitch'/></filter><rect width='100%25' height='100%25' filter='url(%23n)' opacity='0.4'/></svg>");
  opacity: 0.025;
  mix-blend-mode: multiply;
}
```

**暖 shadow 体系**（替代默认 drop shadow）：

```css
/* PaperCard 基础阴影 — 暖色，不是 cool blue 的 box-shadow */
--shadow-paper: 0 1px 2px var(--warm-shadow-1), 0 4px 12px var(--warm-shadow-2);
/* Editorial 开篇大 card */
--shadow-chapter: 0 2px 4px var(--warm-shadow-1), 0 12px 40px var(--warm-shadow-2);
```

### Editorial Decorations（v2.0 — 新增 4 个原子组件）

1. **DropCap**（首字母放大）
   - 用于每章 body 首段第一字
   - 中文：首字下沉 3 行，Noto Serif SC，weight 500，color `var(--chestnut)`
   - 英文：Source Serif 4，float left

2. **PullQuote**（引文）
   - 用于段落中段插入金句 / 用户原话
   - `var(--fs-quote)`，Source Serif 4 italic，color `var(--chestnut)`
   - 左侧无色条（禁 impeccable），用上下 `--line` 细线 + 斜体区分

3. **Kicker**（章节导语小字）
   - 在 Chapter hero 上方，sans-serif uppercase，tracking 0.2em
   - color `var(--chestnut)`，11px
   - 例："CHAPTER III · THE TURNING POINT"

4. **ChapterOpener**（章节开篇大号罗马数字 + 装饰线）
   - 每章开头一个装饰层：大号罗马数字（衬线体 120-160px）作为 watermark 背景
   - 顶部一条 `var(--line)` 水平装饰线
   - 首段前留 48-64px 垂直留白

### 排版规则（v2.0 更严）

- 中文 line-height：base + 0.1（已定）
- 段落最大宽度：**65-72ch**（long form 阅读最优，比 v1 的 65-75ch 更严）
- 字号比例：**1.3-1.4 ratio**（比 v1 的 1.25 更强对比）
- Page padding：移动 24px / 桌面 **96-128px**（长 form 比 GrowthLog 的 64px 更宽）
- 每章之间纵向呼吸：**128-160px**（v1 只有 py-16/24，不够）

### 视觉效果（v1 继承 + v2.0 强化）

**不要**：glassmorphism、cyan/purple gradient、border-left 色条、gradient text、large icon-above-heading 模板感

**要**：
- ✅ Noise overlay（v2.0 必达）
- ✅ 暖 shadow（v2.0 替代 cool shadow）
- ✅ 章节分隔线 + 装饰性罗马数字
- ✅ DropCap / PullQuote / Kicker / ChapterOpener 装饰组件
- ✅ Hero 区极致大字（72-96px）

### 主题

**浅色（默认且唯一）** — 温暖与浅色协同。深色模式留作未来扩展，v2.0 不做。

## Page Editorial Intensity（v2.0 新增 — 每页 editorial 强度分级）

v2 全前端不是所有页都一视同仁地"editorial"。按"用户停留时长 + 长 form 比例"分三级：

| 页面 | 强度 | 特征 |
|---|---|---|
| **ReportPage** | 🌟🌟🌟 最强（like reading a feature article） | 满配 DropCap / PullQuote / Kicker / ChapterOpener，段落宽 65ch，章节留白 128px+，Hero 96px |
| **ProfilePage** | 🌟🌟🌟 强（like filling a journal） | Kicker + ChapterOpener；表单页也保留编辑感，不做成 Bootstrap form |
| **HomePage** | 🌟🌟 中（like magazine cover） | Hero 级 display；信息密度低，大留白 |
| **CoachResultPage** | 🌟 弱（聊天克制 editorial） | 保留字体和色板，但**不要 DropCap/PullQuote**（聊天节奏快） |
| **GrowthLogPage / Explorer / Detail** | 🌟🌟 中 | Chapter 节奏；但以展示数据为主，editorial 装饰节制使用 |

**原则**：**装饰是温度的手段，不是目的**。越是长 form 阅读（Report）越重；越是快节奏交互（Coach 聊天）越轻。

## Motion（v2.0 新增）

v1 未定义，v2 全前端统一：

- **Entrance**：页面进入按 chapter stagger fade-in（每章延迟 80-120ms，opacity 0→1 + translateY 12→0）
- **Scroll**：长 form 页支持 scroll-triggered DropCap pulse / PullQuote fade-in
- **Interaction**：hover 用 color transition（150ms ease-out）而非 scale，editorial 不该"跳"
- **Easing**：ease-out-quart / ease-out-expo（不用 bounce/elastic）
- **Reduced-motion**：`prefers-reduced-motion: reduce` 时全部关闭 staggered entrance，保留 fade

## Design Principles（v1 继承，不变）

1. **温度优先于精确** — 数字诚实给（覆盖率 38%），但语言软化（"已经走完三分之一"）
2. **共情前置于分析** — 用户每个状态先被命名再被分析
3. **零评判用语** — 删 "应该 / 必须 / 建议你要 / 你需要"
4. **章节式叙事** — 页面像翻书的章节，不是看仪表盘
5. **留白即温度** — 拒绝把屏幕填满

v2.0 新增原则：
6. **装饰为温度服务** — DropCap / PullQuote 只在让用户"感到在读一封信"时才加，不为装饰而装饰
7. **阅读节奏分级** — Report 是长 form，Coach 是对话，**不同页面不同强度**，不要把所有页做成同一套模板

## Constraints（v2.0 更新）

### 技术栈（v2 子项目）

- **v2 前端**：`frontend-v2/`（新开 P2 子项目，完全隔离）
- **技术选型**：Vite + React 19 + TypeScript + Tailwind v4 + shadcn/ui + Radix + framer-motion
- **字体**：Google Fonts（Bricolage Grotesque / Source Serif 4 / Noto Sans SC / Noto Serif SC / JetBrains Mono），不引入新 npm 字体依赖
- **API**：复用现有 backend API（http://localhost:8000），API client 独立拷贝一份避免与 v1 耦合

### 规范（必达）

- WCAG AA — 文字对比度 / focus ring / reduced-motion
- 中文主语言 — line-height +0.1
- 现有 backend API 契约不动 — v2 前端消费但不改后端
- 不引入新 npm 字体依赖

### 页面实施顺序（用户已拍）

Report → Profile → Home → Coach → RoleDetail / Graph / Pursuits / ProjectDetail
GrowthLog v2 最后迁移（v1 已落地，冻结）

## Differentiation

**用户离开时记住的不是"用了一个 AI 工具"，而是"被一个懂我的学长聊了几句"。**

v2.0 更强的差异化：
- 所有同类产品（Boss / 智联 / 小猿 / 简单心理 等）都是 SaaS / 教育产品 / 咨询工具感
- 我们是**给 22 岁迷茫学生的 editorial letter** — 读起来像一期为他定制的杂志，每一章被一句话戳中

如果某个 design 决策让产品看起来更像"SaaS 工具"或"教育平台"或"商业应用"，回到本文档对照。
