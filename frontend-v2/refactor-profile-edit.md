# Profile 编辑模式重构问题清单

## 1. 视觉风格严重分裂
- **只读模式**：杂志风（衬线标题、暖米色 `#F9F4EE`、细线边框、大量留白）
- **编辑模式**：表单风（绿色技能标签、棕色大按钮、四列密集网格、冷色 UI）
- 两者像两个产品，用户从只读页点击"编辑档案"后视觉断层极大

## 2. 布局密度不合理
- 编辑模式采用 `grid-cols-4` 四列布局，在 1200px 容器内塞入教育、技能、软技能、目标四个区块
- 技能标签区域（绿色 pill）拥挤且配色和全局暖色调冲突
- SJT "开始测试"按钮竖排文字，视觉突兀

## 3. 功能冗余 vs 只读页缺失
- **编辑模式有、只读模式没有的功能**：
  - 技能标签删除（× 按钮）
  - 知识领域管理
  - SJT 情境判断测试入口
  - 实习/项目 inline 编辑表单
  - 推荐方向列表（"让 AI 帮我推荐"/"我去图谱探索"）
- **只读模式有、编辑模式没有的功能**：
  - AI 摘要横幅
  - 岗位匹配摘要卡片
  - 技能结构/能力差距表格
  - 经历时间线

## 4. 交互模式不一致
- 旧版前端编辑入口在左侧 sidebar，点击后右侧内容区切换为 `ManualProfileForm` 大表单
- 新版前端采用整页切换（`isEditing` state），从只读页完全跳转到另一个页面结构
- 用户预期：在只读页上直接 inline 编辑或弹窗编辑，而不是进入另一个世界

## 5. 目标岗位设定流程断裂
- 只读页无目标时：有"设定目标岗位"按钮 → 滚动到本页 `#recs-section` 推荐区块
- 但推荐数据常为空，显示"推荐方向还在生成中"占位
- 编辑模式"目标与推荐"区块的两个按钮（"让 AI 帮我推荐"/"我去图谱探索"）已 wire up，但和只读页不在同一流程

## 6. "重新生成画像"需求未对齐
- 后端有 `POST /profiles/reparse` 接口（重新解析简历 raw_text 生成画像）
- 旧版前端 `useProfileData` 定义了 `handleReparse` 但**无任何页面调用**（死代码）
- 新版前端暂无 UI 入口，用户是否需要此功能、按钮文案和位置待确认

---

## 建议重构方向（供参考）

### 方向 A：Inline 编辑 + 弹窗
- 保留只读页为唯一主视图
- 每个可编辑区块（教育、技能、经历、目标）加右上角编辑图标
- 点击后弹出模态框编辑，模态框视觉风格和只读页一致
- 删除整页"编辑模式"

### 方向 B：编辑页视觉对齐
- 保留当前功能结构，但全部换成只读页风格
- 绿色 pill 标签 → 棕/灰色系
- 四列 → 宽松的两/三列
- 衬线标题 + 暖米色背景 + 细线边框
