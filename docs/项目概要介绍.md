## 1. 项目概览

### 1.1 项目定位

面向计算机类及信息化相关专业大学生的 **AI 职业规划助手**，通过大语言模型、岗位知识图谱和 **Anthropic Economic Index (AEI)** 权威数据，为学生提供能力画像构建、人岗匹配诊断和个性化职业发展报告，实现从"认识自己"到"精准规划"的完整闭环。

### 1.2 项目背景

当前高校学生在职业规划中普遍面临四大痛点：

1. **自我认知模糊**：缺乏对自身兴趣、能力、性格的深度剖析，所做规划缺乏个性化和可行性；
2. **职业信息不对称**：对行业和岗位的了解多来自碎片化渠道，难以区分"热门噱头"与"真实需求"；
3. **外部指导缺位**：高校生涯课程以理论为主，专业师资匮乏，难以提供贴合实际的个性化建议；
4. **规划落地性差**：职业规划停留在纸面，缺乏动态调整能力，遭遇挫折易陷入迷茫。

本系统以 **150 万+条真实招聘数据 + 409 MB Anthropic Economic Index 数据** 为基座，通过 ETL 管线与 LLM 富化构建 45 节点岗位知识图谱，在此之上搭建多智能体架构，为学生提供可操作、可解释的职业规划服务。

### 1.3 系统设计框架

系统采用四层架构，前后端分离，AI 能力由独立的 Agent 层提供：

```
┌───────────────────────────────────────────────────┐
│  前端展示层（React 19 + TypeScript + @xyflow/react）│
│  画像 / 图谱 / JD诊断 / 报告 / 成长档案 / AI 教练 / 模拟面试 │

└──────────────────────┬────────────────────────────┘

┌──────────────────────▼────────────────────────────┐
```

| 至少使用**一个**大语言模型 | ✅ 通义千问 qwen-plus / qwen-max **全流程覆盖** | 画像提取、匹配分析、报告生成、教练对话、模拟面试 |
| 可操作性、可解释性 | ✅ 图谱节点绑定防幻觉 + evidence 字段引用具体数据 | LLM 输出受事实约束 |

### 四维人岗匹配算法（展开说明）

| 维度 | 权重 | 评分依据 |
|------|-----|---------|
| **职业技能** | 50% | 技能名称匹配 + 熟练度加权（高级 1.0 / 中级 0.75 / 熟悉 0.5 / 了解 0.25） |
| **发展潜力** | 25% | 项目数量与质量 + 获奖经历 + 学历层次 |
| **基础要求** | 15% | 学历达标度(40%) + 专业相关度(35%) + 经验匹配(25%) |
| **职业素养** | 10% | SJT 测评五维度与岗位要求的对比分析 |

**综合评分公式：** `总分 = Σ (维度分 × 维度权重)`

同时，系统通过 **text-embedding-v3** 进行语义级技能匹配，解决"简历写 Vue 但岗位要求前端框架"等同义表述问题，余弦相似度阈值 ≥ 0.65 视为匹配。

## 3. 核心创意

市面上的"AI 简历/职业规划"产品普遍存在三个根本问题：**LLM 一本正经编造岗位信息（幻觉）**、**建议空泛不具操作性**、**AI 代写的简历一到面试就穿帮**。更深层的问题是：市面产品给出的"AI 影响评估"几乎都靠业内自媒体估算，没有权威数据兜底。本项目以"**数据 · 架构 · 机制**"三位一体的设计解决这些问题：

- **创意一：AEI 权威数据接入** —— 系统本地部署完整 **Anthropic Economic Index (409 MB)**，包含 4 个时间版本的职业级 AI 暴露度、任务级 AI 渗透率、自动化 vs 增强六维分解，通过 text-embedding-v3 完成中文岗位任务 → O\*NET 英文任务的语义对齐，让每个岗位的"AI 替代压力"都有权威数字支撑。

- **创意二：数据驱动的图谱锚点** —— 关键结论性输出（方向对齐、岗位推荐等）必须引用图谱中已存在的 `node_id` 与 `evidence` 字段，否则被后端直接过滤；其余软性输出通过 prompt 层 evidence 约束 + 分级人工抽检，从架构层面阻断幻觉。

- **创意三：多智能体 + 三级意图识别** —— 6 个领域专家 Agent 分工协作，意图识别三级分类（正则 → 语义 → LLM 兜底）—— 日常高频意图由前两级直接命中，仅复杂长句回退到 LLM 分类，显著降低延迟与成本。Coach Agent 额外装载 13 个 Skill，采用 Progressive Disclosure 模式，相比全量 push 节省 ≥ 60% token。

- **创意四：画像联动的 AI 模拟面试** —— 模拟面试不再出通用八股题，而是**自动读取成长档案中的真实项目**、**目标方向的 gap 技能**、**发展报告的能力评估结论**，让每道题都围绕学生的实际情况。支持 6 个技术方向（C++/前端/Java/算法/产品/测试）、自定义题量与题型、历史弱项自动复习、空答案强制 0 分。

- **创意五：AI 诊断但不代写** —— AI 只负责识别空洞模式和提供格式范本，填空必须由学生亲自完成（前端硬校验数字+长度+动作动词），避免面试穿帮。

## 4. 功能与特色

### 4.1 核心闭环与功能模块

系统围绕"**画像 → 匹配 → 报告 → 成长 → 面试**"的核心闭环，提供七大功能模块：

| 模块 | 功能说明 |
|------|---------|
| **能力画像** | 上传简历（PDF/图片/粘贴）→ LLM 结构化提取 → SJT 情景测评软技能 → 自动定位图谱坐标 |
| **岗位图谱** | 45 节点可视化探索 · 晋升路径（5 级）· 换岗路径（101 条）· 技能要求 · **AEI 驱动的 AI 影响分析** · 典型项目 |
| **JD 诊断** | 粘贴真实 JD → 四维匹配评分 → 技能差距清单 → top 缺口高亮 → 历史对比 → **图谱关联分析（AI 替代压力 / 转岗路线）** |
| **发展报告** | AI 综合评价 + 方向对齐分析 + 三段式补法地图 + 个性化行动计划 + 一键导出 |
| **成长档案** | 项目追踪 / 求职追踪 / 档案精修三合一，持续记录学生成长轨迹 |
| **AI 模拟面试** | **6 方向 Skill 驱动** · **画像联动出题**（成长项目/gap 技能/报告结论）· **历史弱项复习** · **自定义题量/题型** · 空答案强制 0 分 · 自动落入成长档案 |
| **AI 教练** | 右侧常驻对话面板，13 个专业 Skill 覆盖求职全场景 |

**核心闭环流程：**

```
上传简历 → 建立能力画像 → 选定目标岗位 → JD 诊断找缺口
                                              ↓
                              生成 AI 发展报告（评价 + 对齐 + 行动计划）
                                              ↓
                  记录项目 / 投递 / 面试 → 模拟面试检验 → 档案精修 → 更新报告（Delta 对比）
```

### 4.2 画像联动的 AI 模拟面试（创新亮点）

本系统的 AI 模拟面试与市面产品的根本区别在于：**题目不是通用八股文，而是围绕学生的真实画像动态生成**。

**出题逻辑：**

```
学生画像（简历技能）
    + 成长项目（ProjectRecord：技术栈、状态、反思）
    + 目标方向 gap 技能（CareerGoal：需要补什么）
    + 发展报告结论（Report：能力评估、技能覆盖）
    + 历史面试弱项（InterviewRecord：skill_gaps）
    + JD 要求（可选）
    ──────────────────────────────────────────────
    → LLM 基于上述上下文生成个性化面试题
```

**具体能力：**

| 特性 | 说明 |
|------|------|
| **6 个方向 Skill** | C++ 系统开发、前端开发、Java 后端、算法、产品经理、测试开发，每个方向有独立的面试官人设 + 分类权重 + 参考知识库 |
| **画像联动** | 自动读取成长档案中的项目、目标 gap 技能、报告结论注入 prompt |
| **历史弱项复习** | 自动查询过往面试的 skill_gaps，下次优先覆盖 |
| **自定义题量/题型** | 题量 3/5/10 题可选，技术/场景/行为题占比可调 |
| **公平评分** | 空答案/敷衍答案 → 后端强制 0 分，LLM 不能给同情分 |
| **反幻觉约束** | 30+ 种占位符检测 + 无项目名时禁止编造 + 自动 fallback |
| **模块链路打通** | 岗位图谱页一键跳转、JD 诊断页带着 JD 出题、面试结果自动落入成长档案 |

### 4.3 岗位知识图谱（45 节点 · 20 岗位族）

以 **150 万+条真实招聘数据 + 409 MB Anthropic Economic Index** 为原料，经完整数据工程管线构建出的岗位知识图谱包含 **45 个岗位节点、101 条转换边**：

**数据资产与构建管线：**

| 数据来源 | 规模 | 在管线中的作用 |
|---------|------|---------------|
| 智联招聘数据库（2016-2025） | ~100 万条 | ETL 分类 → 技能频率 · 市场趋势 · 薪资分布 |
| 上市公司招聘大数据（2014-2026） | ~50 万条 | 补充数据空白 + 行业分布信号 |
| 企业命题提供岗位数据 | 9,959 条 | 补充验证 + 岗位描述语料 |
| **Anthropic Economic Index (AEI)** | **409 MB** | **权威 AI 影响评估**：职业级暴露度 + 任务级渗透率 + 自动化/增强六维分解 |
| O\*NET 任务框架 | 20000+ 任务 | AEI 与中文岗位任务的语义对齐锚点 |

**图谱节点字段（100% 覆盖率）：**

- **技能分层**：core / important / bonus 三级，每项带出现频率
- **晋升路径**：每节点 5 级职业阶梯
- **软技能画像**：沟通、学习、抗压、创新、协作五维度
- **项目推荐**：2-3 个典型实战项目 + 推荐理由
- **AI 影响叙事**：基于 AEI 真实数据的冲击与机遇分析
- **市场指标**：替代压力、薪资中位数、人机协作杠杆

### 4.4 AEI 驱动的 AI 冲击层（本项目独有）

市面上的职业规划产品评估"这个岗位会不会被 AI 替代"时，多数靠业内自媒体估算或简单 prompt 询问 LLM。本系统完全不同：**所有 AI 冲击评分都基于 Anthropic Economic Index 的真实数据**。

**AEI 数据简介**：Anthropic 官方在 HuggingFace 公开的权威数据集，基于**数百万条真实 Claude 对话**聚合而成，精确刻画 AI 在现实经济中的任务级渗透和职业级暴露。本系统本地部署完整数据（`data/EconomicIndex/`）。

**六维分解含义**：

| 维度 | 含义 |
|------|------|
| directive | 指令式自动化 —— 用户让 AI 全权执行 |
| feedback_loop | 反馈循环自动化 —— AI 执行、人类校验、AI 修订 |
| task_iteration | 迭代增强 —— 人类主导，AI 协助迭代 |
| validation | 验证辅助 —— AI 帮人类检查 |
| learning | 学习辅助 —— AI 帮人类学习 |
| filtered | 无法分类 |

前 2 列偏自动化（AI 替代人），后 3 列偏增强（人+AI 协同）。

**在系统里的三个核心评分**：

- **`replacement_pressure`**（AI 替代压力 0-100）
- **`human_ai_leverage`**（人+AI 协同杠杆 0-100）
- **`zone`**（战略定位）—— leverage / transition / caution / critical 四象限

### 4.5 三段式技能补法地图

不同于简单的"缺什么补什么"，系统将技能缺口按**最佳学习路径**分为三类：

| 类别 | 典型技能 | 建议路径 |
|------|---------|---------|
| 📚 系统学习型（learn） | 数据结构、操作系统、算法 | 在线课程 / 教材 / 刷题 |
| 🛠️ 项目实践型（practice） | 高并发、分布式、性能优化 | 做项目才能真正掌握 |
| 🔀 先学后做型（both） | Docker、K8s、Git | 先学概念，再做项目巩固 |

报告底部诚实统计："N 个缺口 · X 个可通过项目补 · (N-X) 个需系统学习"，帮助学生合理安排投入。

### 4.6 档案精修机制——AI 诊断但不代写

档案精修是本系统最反套路的设计。区别于市面上"AI 一键优化简历"的做法，本系统坚持"AI 诊断 + 学生填写"的原则：

- **AI 诊断**：自动识别 4 类空洞模式——缺量化数字、缺具体成果、描述过短（<30 字）、只说"参与"未说明"负责"
- **格式范本**：为每条待完善项展示参考格式
- **学生填写**：前端硬校验——必须包含数字 + 长度 ≥ 30 字 + 包含动作动词，不达标按钮禁用
- **闭环验证**：下一次报告生成时，已补充项目自动标记"通过"
