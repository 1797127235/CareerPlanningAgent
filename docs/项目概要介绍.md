## 1. 项目概览

### 1.1 项目定位

面向计算机类及信息化相关专业大学生的 **AI 职业规划助手**，通过大语言模型与岗位知识图谱，为学生提供能力画像构建、人岗匹配诊断和个性化职业发展报告，实现从"认识自己"到"精准规划"的完整闭环。

### 1.2 项目背景

当前高校学生在职业规划中普遍面临四大痛点：

1. **自我认知模糊**：缺乏对自身兴趣、能力、性格的深度剖析，所做规划缺乏个性化和可行性；
2. **职业信息不对称**：对行业和岗位的了解多来自碎片化渠道，难以区分"热门噱头"与"真实需求"；
3. **外部指导缺位**：高校生涯课程以理论为主，专业师资匮乏，难以提供贴合实际的个性化建议；
4. **规划落地性差**：职业规划停留在纸面，缺乏动态调整能力，遭遇挫折易陷入迷茫。

本系统以 150 万+条真实招聘数据为基座，通过 ETL 管线与 LLM 富化构建 45 节点岗位知识图谱，在此之上搭建多智能体架构，为学生提供可操作、可解释的职业规划服务（数据与构建细节详见 §4.3）。

### 1.3 系统设计框架

系统采用四层架构，前后端分离，AI 能力由独立的 Agent 层提供：

```
┌───────────────────────────────────────────────────┐
│  前端展示层（React + TypeScript + @xyflow/react）   │
│  画像 / 图谱 / 报告 / 成长档案 / AI 教练           │
└──────────────────────┬────────────────────────────┘
                       │ REST / SSE
┌──────────────────────▼────────────────────────────┐
│  后端服务层（FastAPI + SQLAlchemy + SQLite）         │
│  12 Router · 13 Service · 四维匹配算法              │
└──────────────────────┬────────────────────────────┘
                       │
┌──────────────────────▼────────────────────────────┐
│  AI Agent 层（LangGraph 多智能体）                   │
│  Supervisor 路由 → 6 专家 Agent → 13 Coach Skill    │
└──────────────────────┬────────────────────────────┘
                       │
┌──────────────────────▼────────────────────────────┐
```

### 四维人岗匹配算法（展开说明）

对齐命题要求的"4 个维度能力分析"，本系统的匹配算法如下：

| 维度 | 权重 | 评分依据 |
|------|-----|---------|
| **职业技能** | 50% | 技能名称匹配 + 熟练度加权（高级 1.0 / 中级 0.75 / 熟悉 0.5 / 了解 0.25） |
| **发展潜力** | 25% | 项目数量与质量 + 获奖经历 + 学历层次 |
| **基础要求** | 15% | 学历达标度(40%) + 专业相关度(35%) + 经验匹配(25%) |
| **职业素养** | 10% | SJT 测评五维度与岗位要求的对比分析 |

**综合评分公式：** `总分 = Σ (维度分 × 维度权重)`

同时，系统通过 **text-embedding-v4** 进行语义级技能匹配，解决"简历写 Vue 但岗位要求前端框架"等同义表述问题，余弦相似度阈值 ≥ 0.65 视为匹配。

## 3. 核心创意

市面上的"AI 简历/职业规划"产品普遍存在三个根本问题：**LLM 一本正经编造岗位信息（幻觉）**、**建议空泛不具操作性**、**AI 代写的简历一到面试就穿帮**。本项目以"**数据 · 架构 · 机制**"三位一体的设计解决这三个问题：

- **创意一：数据驱动的图谱锚点**——关键结论性输出（方向对齐、岗位推荐等）必须引用图谱中已存在的 `node_id` 与 `evidence` 字段，否则被后端直接过滤；其余软性输出通过 prompt 层 evidence 约束 + 分级人工抽检，从架构层面阻断幻觉。
- **创意二：多智能体 + 三级意图识别**——6 个领域专家 Agent 分工协作，意图识别三级分类（正则 → 语义 → LLM 兜底）——日常高频意图由前两级直接命中，仅复杂长句回退到 LLM 分类，显著降低延迟与成本。
- **创意三：三段式技能补法地图**——将技能缺口按 `learn / practice / both` 三类学习路径分类，打破"缺什么补什么"的粗放思路，帮学生在有限时间内精准发力。
- **创意四：AI 诊断但不代写**——AI 只负责识别空洞模式和提供格式范本，填空必须由学生亲自完成（前端硬校验数字+长度+动作动词），避免面试穿帮。

四个创意的具体实现与数据支撑，详见 §4「功能与特色」。

## 4. 功能与特色

### 4.1 核心闭环与功能模块

系统围绕"**画像 → 匹配 → 报告 → 成长**"的核心闭环，提供六大功能模块：

| 模块 | 功能说明 |
|------|---------|
| **能力画像** | 上传简历（PDF/图片/文字）→ LLM 提取结构化数据 → SJT 情景测评软技能 → 自动定位图谱坐标 |
| **岗位图谱** | 45 个岗位节点可视化探索，含晋升路径（5 级）、换岗路径、技能要求、AI 影响分析 |
| **JD 诊断** | 粘贴真实 JD → 四维匹配评分 → 技能差距清单 → top 缺口高亮 |
| **发展报告** | AI 综合评价 + 方向对齐分析 + 三段式补法地图 + 个性化行动计划 + 一键导出 |
| **成长档案** | 项目追踪 / 求职追踪 / 档案精修三合一，持续记录学生成长轨迹 |
| **AI 教练** | 右侧常驻对话面板，13 个专业 Skill 覆盖求职全场景，支持语音交互 |

**核心闭环流程：**

```
上传简历 → 建立能力画像 → 选定目标岗位 → JD 诊断找缺口
                                              ↓
                              生成 AI 发展报告（评价 + 对齐 + 行动计划）
                                              ↓
                  记录项目 / 投递 / 面试 → 档案精修 → 更新报告（Delta 对比）
```

### 4.2 LangGraph 多智能体协作架构

系统采用 **Supervisor Pattern**，由一个路由 Agent 统一调度 6 个专家 Agent：

| Agent | 职责 | 典型能力 |
|-------|------|---------|
| navigator | 岗位图谱探索 | 搜索节点、获取晋升路径、比较岗位 |
| profile | 画像诊断 | 画像摘要、图谱定位、能力评估 |
| jd | JD 诊断 | 四维匹配评分、技能差距分析 |
| growth | 成长追踪 | 项目进展、面试记录、学习状态 |
| coach | 教练对话 | 13 个专业 Skill 覆盖求职全场景 |
| search | 信息搜索 | 实时岗位搜索、招聘信息获取 |

**意图识别采用三级分类机制**，在保证准确率的同时兼顾响应速度：正则表达式处理高频确定意图（如 JD 文本检测、简短确认），语义路由通过 embedding 相似度匹配中频意图，仅当前两级无法判断时才触发 LLM 分类兜底。

### 4.3 岗位知识图谱（45 节点 · 20 岗位族）

以 **150 万+条真实招聘数据**为原料，经完整数据工程管线构建出的岗位知识图谱包含 **45 个岗位节点、101 条转换边**：

**数据资产与构建管线：**

| 数据来源 | 规模 | 在管线中的作用 |
|---------|------|---------------|
| 智联招聘数据库（2016-2025） | ~100 万条 | ETL 分类 → 技能频率 · 市场趋势 · 薪资分布 |
| 上市公司招聘大数据（2014-2026） | ~50 万条 | 补充 2019-2020 数据空白 + 行业分布信号 |
| 企业命题提供岗位数据 | 9,959 条 | 补充验证 + 岗位描述语料 |
| O\*NET / Anthropic 经济指数 | 409 MB | 校准 AI 替代压力 · 人机协作杠杆 |

**图谱构建流程（4 阶段 ETL + 8 步富化）：**

```
原始招聘数据(150万+) → ① LLM分类(20岗位族) → ② 时序聚合(年度趋势)
→ ③ 市场信号生成(需求/薪资/AI渗透) → ④ 技能频率统计(core/important/bonus)
→ ⑤ 节点构建(45节点) → ⑥ 晋升路径生成(L1-L5) → ⑦ 转换边生成(101条)
→ ⑧ O*NET校准(替代压力/AI杠杆) → ⑨ LLM富化(市场洞察/项目推荐)
→ ⑩ 软技能计算(5维度) → ⑪ 区分特征标注 → ⑫ 质量校验(去污染)
→ 最终 graph.json (45节点 · 101边 · 281KB)
```

**图谱节点字段（100% 覆盖率）：**

- **技能分层**：core / important / bonus 三级，每项带出现频率
- **晋升路径**：每节点 5 级职业阶梯（如：初级前端工程师 → 前端工程师 → 高级前端工程师 → 前端技术专家 → 前端架构师）

**换岗路径示例：**

| 起始岗位 | 换岗方向 1 | 换岗方向 2 | 换岗方向 3 |

### 4.4 三段式技能补法地图

不同于简单的"缺什么补什么"，系统将技能缺口按**最佳学习路径**分为三类：

| 类别 | 典型技能 | 建议路径 |
|------|---------|---------|
| 📚 系统学习型（learn） | 数据结构、操作系统、算法 | 在线课程 / 教材 / 刷题 |
| 🛠️ 项目实践型（practice） | 高并发、分布式、性能优化 | 做项目才能真正掌握 |
| 🔀 先学后做型（both） | Docker、K8s、Git | 先学概念，再做项目巩固 |

报告底部诚实统计："N 个缺口 · X 个可通过项目补 · (N-X) 个需系统学习"，帮助学生合理安排投入。

### 4.5 档案精修机制——AI 诊断但不代写

档案精修是本系统最反套路的设计。区别于市面上"AI 一键优化简历"的做法，本系统坚持"AI 诊断 + 学生填写"的原则：

- **AI 诊断**：自动识别 4 类空洞模式——缺量化数字、缺具体成果、描述过短（<30 字）、只说"参与"未说明"负责"
- **格式范本**：为每条待完善项展示参考格式（如："实现了 XX 模块，QPS 从 Y 提升到 Z"）
- **学生填写**：前端硬校验——必须包含数字 + 长度 ≥ 30 字 + 包含动作动词（负责/主导/实现），不达标按钮禁用
- **闭环验证**：下一次报告生成时，已补充项目自动标记"通过"

**设计哲学**：AI 代写的简历一到面试就穿帮，强迫学生用真实数据填空，才是真正有指导意义的职业规划。
