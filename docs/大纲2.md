P2 开场引入：为什么很多学生越努力越迷茫
标题
很多大学生缺的不是努力，而是方向

核心内容

不少学生并不是不想准备求职，而是不知道该往哪里准备
看了很多岗位、学了很多内容，却依然无法形成清晰职业路径
简历、测评、投递、面试、复盘彼此割裂，努力无法沉淀
结果是：准备很多，决策很少；投入很多，反馈很弱

一句总结
职业规划最大的痛点，不是"信息不够"，而是"没有形成连续的决策与成长路径"。

版式建议

· 底部总结句用 Coral 色加粗，与正文拉开层次
· 整页像电影开场旁白——安静、克制、让评委读进去

P4 现有方案为什么没有真正解决问题
标题
工具很多，但大多只解决了职业规划中的一个环节

核心内容

职业测评（MBTI / 霍兰德）：帮学生认识偏好，但无法对应到真实岗位的技能要求
招聘平台（Boss直聘等）：能看到岗位，但无法结合个人画像做系统判断
简历优化工具：改善表达，但改不了"能力与岗位不匹配"的根本问题
通用大模型（ChatGPT / 文心一言）：能对话，但岗位薪资等事实缺乏依据，容易幻觉
刷题平台（牛客 / 力扣）：题库丰富，但不做画像差距诊断，练完不知道短板在哪

一句总结
每个工具在单点有价值，但缺的是一条贯通"认知→选择→诊断→训练→迭代"的完整闭环。

版式建议
· 5 张等宽横向卡片一字排开，卡片高度统一
· 每张卡片结构：顶部产品名（加粗）→ 中间一句"强在哪"（灰色）→ 底部一句"缺什么"（Coral 色加粗）

第二部分：把方案讲清楚

P5 我的答案：把职业规划做成一个闭环 · 深色重页
标题
职途智析：让职业规划从一次判断，变成持续成长过程

核心内容
我把大学生职业规划拆成 4 个连续阶段，每个阶段的输出是下一阶段的输入：

读懂学生 → 上传简历 + 情景测评，形成结构化能力画像
读懂岗位 → 岗位知识图谱 + 真实招聘数据，理解用人需求和迁移路径
找到差距 → JD 四维诊断 + AI 影响分析，生成可执行补强路径
持续迭代 → 面试训练结果、投递反馈自动回灌档案，下次建议更精准

关键区别
不是 4 个独立功能的并列，而是一条数据贯通的链路——画像驱动诊断，诊断驱动训练，训练反哺画像。

版式建议
· 深色背景（#0F1E3D），这是全场第一个转折点
· 页面中央放一个圆形闭环箭头图：4 个节点沿环排列（读懂学生 → 读懂岗位 → 找到差距 → 持续迭代）
· 每个节点 = 白色 Lucide 图标 + 两字标签，节点间用弧形箭头连接
· 圆环中心放一句话："数据贯通的链路"
· 底部"关键区别"用 Coral 色小字补充，不抢主视觉

P6 为什么这套建议更可信
标题
不是凭感觉推荐，而是基于三类真实数据做判断

核心内容

我的每一条建议背后，都有可追溯的数据来源：

第一类：真实招聘语料
系统接入公开招聘数据集与授权合作数据源，经过去重、岗位族聚类、技能实体抽取，形成百万级结构化语料。学生看到的技能要求和市场趋势，来自真实企业的真实 JD，不是 AI 编的。

第二类：岗位知识图谱
覆盖 IT/CS 领域主要应届可达岗位，每个节点包含：必备技能、加分技能、晋升路径、典型实战项目模板。节点之间有迁移边——系统能告诉学生"从 A 岗转到 B 岗需要补什么、大概多久"。

第三类：AI 时代职业影响数据（AEI）
基于 Anthropic Economic Index 公开数据集，这是目前最权威的任务级 AI 自动化分析之一。我们做了三步本土化映射：
  · AEI 任务标签 → 国内岗位族
  · 任务自动化/增强分值 → 岗位级"AI 替代压力"与"人+AI 协同杠杆"
  · 结合国内招聘语料的技能权重做再校准
学生能看到自己目标岗位在 AI 时代的战略定位（是该拥抱 AI 还是该找 AI 替代不了的方向）。

一句总结
学生得到的不是"好听的建议"，而是有出处、可追问、可验证的行动方向。

版式建议
· 三栏等宽并排，每栏一个数据类型卡片
· 每卡结构：顶部 Lucide 图标（Database / Network / TrendingUp）+ 数据类型名称（加粗）
  → 中间 2-3 行说明文字（正常字重，灰色）
  → 底部一句 Coral 色小字："学生看到的是……"（点明数据如何变成体验）
· 标题行居中，说明文字左对齐

P7 用户旅程：小王的 72 小时
标题
从迷茫到行动，小王在 72 小时内完成第一次职业校准

核心内容

Day 1：上传简历，建立画像
系统从他的简历中提取了项目经历和技能标签，同时发现他缺少容器化（K8s）相关经验——这个空白他自己从没注意过

Day 2：探索图谱，锁定方向
小王不再只看"什么热门"，而是在图谱上看到：从他当前技能基础出发，后端开发和 DevOps 是最现实的两个候选方向，而算法岗需要补的东西太多，暂不建议盲冲

Day 3：诊断 JD + 模拟面试，形成计划
他粘贴了一份真实 JD，系统给出四维匹配分析和技能缺口清单；接着做了一场模拟面试，面试题目直接围绕他的项目经历深挖——暴露的短板自动进入成长档案，下次面试会优先覆盖

一句总结
职业规划第一次不再停留在"我想干什么"，而是走向"我下一步该补什么、怎么补"。

P9 功能模块二：岗位图谱与职业路径
标题
不是只推荐一个岗位，而是帮学生看见一条可走的路

核心内容

图谱覆盖 IT/CS 领域主要应届可达岗位，可视化探索
每个节点包含：核心技能、晋升 5 级路径、AI 影响分析、典型实战项目模板
节点之间有三类关系边：
  · 晋升边（同岗族 L1→L5）
  · 换岗边（跨岗族迁移，附换岗成本评估）
  · 邻接边（技能高重合的相近岗位）
换岗成本由四个因子计算：技能重合度 + 经验可迁移性 + 学习曲线 + 市场需求

我的不同
学生看到的不是"推荐列表"，而是一张有距离感的地图——哪些岗位现在就能够到，哪些需要半年补强，哪些虽然想去但当前跨度太大，系统会诚实告知。

一句总结
职业规划真正重要的，不是"选中一个岗位"，而是"看到自己能怎么走过去"。

版式建议
· 左 55%：真实图谱截图（缩略，节点按岗位族染色聚类）

P10 功能模块三：JD 诊断
标题
面对真实岗位，终于知道差距在哪、该先补什么

核心内容

学生粘贴真实 JD → 系统结合画像做四维分析：
  · 基础要求（学历 / 经验门槛达标度）
  · 职业技能（核心技能覆盖 + 熟练度加权）
  · 职业素养（软技能匹配，要求行为证据而非自述标签）
  · 发展潜力（项目复杂度递增 / 竞赛开源贡献等可验证事实）
输出：技能差距清单（高优先级标红）+ AI 影响分析 + 推荐转岗路线

我的不同
评分有严格的反通胀机制——素养维度不因"标签数量多"就给高分（标签是自述不是证据），潜力维度有基准分上限（领域不相关的技能不算迁移优势）。系统宁可给一个准确的低分，也不给一个让人自我感觉良好的虚高分。

一句总结
很多人不是不适合岗位，而是不知道自己该先补什么——这份诊断给的是优先级。

版式建议
· 上半页：四维雷达图或四个横向条形图（基础 / 技能 / 素养 / 潜力），简洁不花哨
· 下半页左右分栏：左栏"已具备"（绿色对勾图标列表）/ 右栏"待补强"（Coral 色感叹号列表，高优先级加粗）
· 页底一行浅灰底条："推荐替代方向 →"后跟 1-2 个岗位标签
· 底部"我们的不同"用小字说明评分机制的严谨性

P12 功能模块五：模拟面试
标题
不是千篇一律的八股，而是围绕你的真实情况出题

核心内容

已上线 6 个方向：C++ 系统开发 / 前端 / Java 后端 / 算法 / 产品 / 测试
每个方向有独立的面试官人设、题目分类权重和参考知识库，题目由 LLM 动态生成

题目从哪来（5 个输入源 → LLM → 个性化题目）
  · 学生的成长项目——围绕真实项目深挖技术细节
  · 目标方向的 gap 技能——优先考察薄弱项
  · 发展报告结论——出题深度与能力评估一致
  · 历史面试弱项——上次暴露的短板这次一定会再问
  · JD 原文要求——贴合真实岗位需求

保障公平
  · 空答案或敷衍回答 → 强制 0 分（后端硬约束，LLM 不能给同情分）
  · 30+ 种占位符模式检测——"XX 公司""某个项目"这类空话直接拦截
  · 面试结果中的 skill_gaps 自动落入成长档案，下次面试优先覆盖

一句总结
模拟面试不只是练回答，而是在帮学生完成一次次精准的能力校准。

版式建议
· 上半页横排三张卡片:
  ① 题目卡片（模拟一道面试题，标注"来自你的项目经历"）
  ② 评分卡片（多维雷达或条形，标注分数 + 一句反馈）
  ③ 档案卡片（标注"skill_gap 自动写入"）
· 三张卡片之间用箭头串联，形成"出题 → 评分 → 回灌"的流程感
· 下半页放"5 个输入源"的简洁示意：5 个小图标一字排开，汇聚到一个 LLM 节点，再输出题目
· 底部"保障公平"用小字补充，不抢主视觉

第四部分：把系统可信度讲清楚

P14 设计哲学：我选择对学生诚实
标题
AI 做观察，不做预言；给工具，不代劳

核心内容
在这个项目里，我主动放弃了一些"看起来很酷"的功能，因为它们对学生有害：

第一条红线：AI 不做预言
系统拒绝输出"你 3 年能到高级""你适合做 X 方向"这类无法验证的预测。只展示事实（技能覆盖了哪些、项目证据有几个）和对齐分析（你的画像跟哪些岗位重合度高），判断权留给学生自己。

第二条红线：AI 不代写
档案精修只给差距诊断 + 格式范本，学生必须用自己真实数据填空。我们宁可画像不完整，也不替学生编造经历——因为模拟面试会追问细节，编的东西一问就穿。

第三条红线：诚实说"我不知道"
当学生数据不足以生成方向对齐时，系统显示"还需要更多数据"并引导补充，而不是用模板填充一份好看但空洞的报告。

一句总结
我相信：一个诚实的系统，比一个讨好用户的系统，对学生更有长期价值。

版式建议
· 三栏等宽并排，每栏用细线框卡片
· 每卡结构：顶部一个大号 Lucide 图标（ShieldX / PenOff / HelpCircle）+ 红线名称（Coral 色加粗）
  → 中间：一句原则（正常字重）
  → 底部：一句具体做法（灰色小字，让评委看到"不是空话"）
· 页底总结句用较大字号居中，Coral 色加粗
· ⚠️ 演讲提示：这页是打动评委的关键页，建议停留 60-90 秒，用具体案例展开讲

P16 技术架构与核心创新
标题
产品背后：不是"套一个大模型"，而是一套完整的工程体系

核心内容

四层系统架构
  前端展示层 → React 19 + Vite + Zustand
  后端服务层 → FastAPI + SQLite + JWT（试点轻量，已预留 PostgreSQL 迁移）
  AI Agent 层 → LangGraph 状态机 + 6 个专家 Agent + 15 个场景 Skill
  大模型与向量层 → 通义千问 + text-embedding-v3（1024 维语义向量）

三个核心工程创新

创新①：多智能体协作，不是一个模型干所有事
6 个专家 Agent 各司其职：画像分析师 / 方向顾问 / JD 诊断师 / 成长教练 / 岗位搜索 / 成长顾问。中央调度器根据意图自动路由，学生感知到的是一个统一的对话入口。

创新②：三级意图识别，平衡成本与精度
  · 第一级：正则快路径——覆盖高频确定性意图（如粘贴 JD、确认语）
  · 第二级：语义向量匹配——用 embedding 相似度兜底模糊表达
  · 第三级：LLM 兜底——只有复杂长文本才调用大模型分类
高频场景走快路径零延迟，长尾场景仍有 LLM 兜底。

创新③：Skill 按需加载，让 AI 注意力更集中
15 个教练场景 Skill 采用 Progressive Disclosure 机制：系统只把 Skill 目录（名称 + 描述）注入上下文，LLM 判断场景后按需加载 1-3 个完整规则。相比全量注入，上下文体积大幅降低，LLM 回复质量更高。

一句总结
真正难的不是"调通一个 API"，而是让 6 个 Agent、15 个 Skill、三级路由在真实对话中协同工作且不出错。

版式建议
· 上 40%：四层架构图，自上而下堆叠（每层一个浅色条带 + 层名称 + 关键技术词）
  层间用向下箭头连接，箭头旁标注通信方式（REST / SSE / Tool Call）
· 下 55%：三栏创新卡片并排
  每卡结构：编号圆圈（①②③）+ 创新标题（加粗）+ 2 行说明（正常字重）
  关键词用 Coral 色标注（如"三级漏斗""Progressive Disclosure"）
· 技术名词只出现在架构图条带里，创新卡片重点讲"为什么这么设计"

P17 技术难点攻关
标题
开发中啃下的三个硬骨头

核心内容

难点一：多源简历解析——学生材料格式是真的混乱
  痛点：文字 PDF / 扫描 PDF / 手机拍照 / 粘贴文本，格式千奇百怪
  常规做法：用一个第三方 SDK，识别率不稳定就认了
  我的决策：三路 fallback
    · 首路：第三方 SDK 处理标准文字版
    · 兜底：自研 LLM 结构化抽取 + 证书/技能正则补全
    · 扫描版：视觉模型多页并行 OCR + 文本清洗
  结果：四种输入格式都能稳定落入标准画像 schema

难点二：AEI 本土化映射——美国数据怎么给中国学生用
  痛点：Anthropic Economic Index 是美国劳动力市场的任务级分析，直接照搬到国内会失真
  常规做法：要么照搬原始数据，要么干脆放弃不用
  我的决策：三步映射 + 招聘语料校准
    · 任务标签 → 国内岗位族
    · 任务自动化/增强分值 → 岗位级替代压力与协同杠杆
    · 结合国内招聘语料的技能权重做再校准
  结果：每个图谱节点都有可引用、可解释的 AI 影响分值

难点三：Agent 意图路由——成本、延迟、精度不可能三角
  痛点：纯 LLM 路由成本高延迟大，纯正则覆盖率低维护累
  常规做法：多数系统"一路 LLM 到底"，简单但贵
  我的决策：三级漏斗（正则 → 语义向量 → LLM）
  结果：高频场景零延迟零成本，长尾场景仍有 LLM 兜底

一句总结
技术选择的目标不是"看起来复杂"，而是让学生在真实使用中获得稳定、可靠的体验。

版式建议
· 三栏等宽并排，每栏一个难点卡片
· 每卡内部四层结构，用不同视觉层级区分:
  ① 痛点（灰色小字 + ⚠️ 图标）
  ② 常规做法（删除线或浅灰底，暗示"不够好"）
  ③ 我的决策（Coral 色加粗，这是视觉重心）
  ④ 结果（绿色对勾 + 一句话）
· 每栏顶部放难点编号 + 短标题（如"简历解析""AEI 映射""意图路由"）
· 页底总结句居中，正常字重
