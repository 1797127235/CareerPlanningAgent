## P3 项目背景：大学生职业规划的四大困境

1. **自我认知模糊** — 选专业看分数不看兴趣，"从众规划"严重
2. **信息不对称** — 对行业了解碎片化，分不清热门噱头与真实需求
3. **指导缺位** — 高校课程重理论、师资脱离一线
4. **落地性差** — 规划停留在纸面，遇挫折全盘否定

**数据锚点**：传统专业毕业生待业率长期徘徊在 23% 以上
来源：《中国教育发展报告》近年度版本（演讲稿中准备好具体年份与出处）

## P4 市面方案局限：为什么现有工具不够

| 工具类型 | 代表产品 | 该环节强在哪 | 缺口 |
|---------|---------|-------------|------|
| 通用求职平台 | Boss 直聘、实习僧 | 岗位撮合、AI 助手答疑 | 无画像诊断、无可落地路径 |
| AI 简历生成器 | 各类一键优化 | 文案润色 | 内容脱离真实经历，模拟面试易穿帮 |
| 职业测评 | MBTI、霍兰德 | 性格与偏好洞察 | 与真实岗位技能脱节 |
| 通用 LLM | ChatGPT、文心一言 | 开放式对话 | 岗位 / 薪资事实缺 evidence，易幻觉 |
| 刷题 APP | 牛客、力扣 | 题库丰富 | 不做画像差距诊断 |

**结论**：每个方案在单点有价值，**但都没有形成"真实数据 → 画像诊断 → 路径规划 → 模拟检验 → 档案迭代"的完整闭环**

## P9 项目特色④：多智能体协作

**6 个专家 Agent**：navigator · profile · jd · coach · growth · search

**三级意图识别**
1. 正则匹配（0ms）— 快路径覆盖高频意图
2. 语义匹配（<50ms）— 向量相似度兜底
3. LLM 兜底 — 复杂长文本才调用大模型

**13 Coach Skill**：采用 Anthropic Skill 官方的 **Progressive Disclosure** 机制
- 按意图 lazy load，每次加载 1-3 个相关 Skill
- 上下文体积相比"全量基线"显著降低
- 短上下文让 LLM 注意力更集中


## P17 技术难点攻关

项目开发中啃下的三个硬骨头，以及我们的工程决策：

**难点①：多源简历解析 · 应对格式大杂烩**
- 痛点：学生简历 = 文字 PDF + 扫描 PDF + 图片 + Word + 粘贴文本，字段千奇百怪
- 市面方案：单一第三方 SDK 对扫描版 / 异形简历识别率低；纯 LLM 直接解析容易漏字段
- 我们的决策：**三路 fallback**
  - 首路：ResumeSDK 处理标准文字版
  - 兜底：自研 LLM 结构化抽取 + 证书 / 技能正则补全
  - 扫描版：qwen-vl-ocr 多页并行 OCR + 文本清洗
- 成果：文字 / 扫描 / 图片 / 粘贴四种输入均能落入标准 schema 入库

**难点②：AEI 本土化映射 · 跨文化数据再校准**
- 痛点：AEI 原始数据是美国劳动力市场任务级分解，国内岗位族不一致、技能权重也有差异
- 市面方案：要么照搬（失真）要么放弃（失去权威数据底牌）
- 我们的决策：**三步映射 + 招聘语料校准**
  - 任务标签 → 国内岗位族
  - 任务分值 → 岗位级替代 / 协同杠杆归一
  - 结合国内招聘语料的技能权重再校准
- 成果：45 节点都有可引用、可解释的 AI 影响分值（P6 四象限即为产出）

**难点③：Agent 意图路由 · 成本-延迟-精度三角**
- 痛点：纯 LLM 兜底成本高 + 延迟大；纯正则覆盖率低 + 维护累
- 市面方案：多数系统是"一路 LLM 到底"
- 我们的决策：**三级意图识别漏斗**
  - 正则快路径（0ms）覆盖高频确定性意图
  - 语义向量（<50ms）兜底模糊表达
  - LLM 最后兜底复杂长文本
- 成果：高频场景响应快且省成本，复杂场景仍有 LLM 保障

**版式提示**：三栏并排卡片，每卡"痛点→市面→决策→成果"四层结构

**步骤**
