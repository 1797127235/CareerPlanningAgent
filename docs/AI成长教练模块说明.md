# AI 成长教练模块说明

## 一句话概述

AI 成长教练是一个**对话式职业发展助手**，通过自然语言交互，根据用户的职业阶段和实时需求，自动匹配对应的场景技能，提供个性化、可落地的指导。

## 核心设计理念

| 设计原则 | 说明 |
|---------|------|
| **场景 Skill 驱动** | 15 个场景化 Skill 自动匹配，不是万能的"闲聊机器人" |
| **Pull-based Context** | 不预加载所有用户数据，而是通过工具按需拉取画像/目标/市场信号 |
| **Progressive Disclosure** | 逐步深入，不一次性输出长篇大论 |
| **阶段匹配** | 根据用户的 career_stage 给出不同引导策略 |
| **数据驱动** | 引用市场数据、画像数据，禁止凭感觉编造 |

## 用户职业阶段

教练根据用户所处的职业阶段，采用不同的引导策略：

| 阶段 | 英文标识 | 典型状态 | 教练侧重 |
|------|---------|---------|---------|
| 探索方向 | `exploring` | 没有明确目标，想了解各个方向 | 方向对比、岗位介绍、推荐分析 |
| 已选目标 | `focusing` | 锁定了目标岗位，开始准备 | 差距分析、技能补齐、项目规划 |
| 求职中 | `job_hunting` | 正在投简历/面试 | 面试复盘、简历优化、心态调整 |
| 冲刺 | `sprinting` | 拿到 offer 在比较，或准备终面 | Offer 对比、最后冲刺、决策支持 |

## 15 个场景 Skill 详解

### 1. coach-greeting — 问候

- **触发**：用户只说了问候语（你好/嗨/hi 等，不超过 10 字）
- **行为**：简单友好问候 + 一个阶段匹配的开放性问题
- **禁止**：不反引用用户画像细节，不调用工具

### 2. coach-direction-scaffold — 方向脚手架

- **触发**：用户表达"没有方向/不知道想做什么"
- **行为**：先问低门槛情境线索，再从线索生成 1-2 个方向种子 + 7 天内可验证的最小动作
- **禁止**：不让用户自己"想想你兴趣是什么"，不给长期规划

### 3. coach-exploring-guide — 探索引导

- **触发**：exploring 阶段 + 用户询问推荐/对比/适合某方向
- **行为**：调用推荐工具拿真实数据，帮用户从推荐方向里选一个设为目标
- **禁止**：不编造系统中不存在的方向名

### 4. coach-decision-socratic — 苏格拉底式决策

- **触发**：用户在多个选项间纠结（"A 还是 B" / "犹豫" / "纠结"）
- **行为**：问价值观问题帮厘清需求，或直接给 2-3 个路径对比表
- **原则**：最多 2 轮反问后必须给具体对比，不让用户永远绕圈

### 5. coach-concern-direct — 担忧直答

- **触发**：用户表达担忧、迷茫、自我怀疑（"我担心" / "怎么办" / "够不够"）
- **行为**：1-2 句共情 + 3-4 句具体判断 + 最多 1 个引导性问题
- **禁止**：不反问"你觉得呢"，直接给判断或信息

### 6. coach-emotional-support — 情绪支持

- **触发**：用户情感宣泄、心理压力、疲惫崩溃（"撑不下去" / "好焦虑" / "想哭"）
- **行为**：先共情，后落地情绪来源，最后给一个今晚就能做的低门槛选择
- **禁止**：不调工具、不追问"你觉得呢"、不空洞鼓励

### 7. coach-comparison-detox — 比较解毒

- **触发**：用户陷入与他人比较（"XX 同学已经拿 offer 了" / "室友晒 offer"）
- **行为**：承认比较的感受 → 拆解信息偏差 → 给低门槛 A/B 选项引回自己能做的事
- **禁止**：不说"每个人节奏不同"/"别跟别人比"这类官话

### 8. coach-confirmation — 确认应答

- **触发**：用户只是简短确认（"好"/"嗯"/"OK"/"收到"）
- **行为**：1-2 句继续推进对话
- **禁止**：不把"好/嗯"当成执行指令去调工具

### 9. coach-request-deliver — 需求交付

- **触发**：用户发出明确具体请求（"帮我 X" / "给我 X" / "分析一下"）
- **行为**：直接交付用户要的内容，必要时先调工具拉数据
- **禁止**：不反问定义或动机

### 10. coach-resume-review — 简历评审

- **触发**：用户请求点评简历（"帮我看看简历" / "我的简历怎么样"）
- **行为**：先调 `get_user_profile` 拿真实数据，从面试官视角给 3 个具体问题 + 1 个最紧迫改进方向
- **禁止**：没调工具就评价等于瞎编

### 11. coach-interview-prep — 面试准备

- **触发**：用户为具体面试求帮助（面前准备/技术复盘/结果复盘）
- **行为**：识别时间窗决定颗粒度 → 调画像预判追问点 → 基于真实项目给具体追问题
- **禁止**：不远水救近火、不空洞鼓励

### 12. coach-market-signal — 市场信号

- **触发**：用户询问某方向的市场前景/薪资/AI 替代风险
- **行为**：调 `get_market_signal` 拿真实数据，基于数据给判断
- **禁止**：不凭感觉说"前景不错"/"AI 会替代"

### 13. coach-profile-builder — 画像构建

- **触发**：用户没有简历，想通过对话建立画像
- **行为**：通过 5-6 个自然问题收集信息（专业→技能→项目→兴趣→意向），确认后保存
- **禁止**：不一次问多个问题、不评判用户水平

### 14. coach-progress-report — 进度汇报

- **触发**：用户主动汇报完成的具体动作（"我把 leetcode hot100 刷完了"）
- **行为**：具体认可 → 把动作从"作业视角"翻译到"求职/能力积累视角" → 给可自测的锚点 → 动态生成 A/B 选择
- **禁止**：不空洞鼓励、不直接打击、不无脑加码

### 15. coach-project-planning — 项目规划

- **触发**：前端项目规划模块触发的 `[项目规划请求]` 标签消息
- **行为**：直接给 8-15 句具体规划，含技术实现 + 分阶段里程碑 + 面试考点 + 能补的技能缺口
- **禁止**：不反问澄清

## 技术架构

```
用户消息
   ↓
Intent Router（语义路由）
   ↓
Supervisor（调度器）
   ↓
Coach Agent（LangChain ReAct Agent）
   ├── 场景 Skill 匹配（15 个 SKILL.md）
   ├── Pull Tools（按需拉取数据）
   │   ├── get_user_profile — 用户画像
   │   ├── get_career_goal — 职业目标
   │   ├── get_market_signal — 市场信号
   │   ├── get_memory_recall — 对话记忆
   │   ├── get_recommended_roles — 推荐岗位
   │   └── save_profile_from_chat — 从对话保存画像
   └── Action Tools（执行操作）
       ├── search_real_jd — 搜索真实 JD
       └── recommend_jobs — 推荐岗位
```

## 与其他模块的关系

| 模块 | 关系 |
|------|------|
| 能力画像 | 教练通过 `get_user_profile` 读取画像数据，通过 `save_profile_from_chat` 写回画像 |
| 岗位图谱 | 教练通过 `get_recommended_roles` 获取推荐，通过 `recommend_jobs` 引导选方向 |
| JD 诊断 | 教练可引导用户去做 JD 诊断，并解读诊断结果 |
| 发展报告 | 教练可参考报告数据给出建议 |
| 软技能评估 | 教练可引导用户完成 SJT 评估，并解读评估结果 |
| 成长档案 | 教练的进度汇报 Skill 与档案追踪联动 |

## 关键文件

| 文件 | 作用 |
|------|------|
| `agent/agents/coach_agent.py` | Coach Agent 定义，ReAct 模式 |
| `agent/skills/coach-*/SKILL.md` | 15 个场景 Skill 的规则定义 |
| `agent/tools/coach_context_tools.py` | Pull-based 上下文工具 |
| `agent/supervisor.py` | 调度器，动态注入 Skill 到 system prompt |
| `agent/intent_router.py` | 语义意图路由 |
| `backend/services/coach_memory.py` | 对话记忆服务 |
| `backend/routers/chat.py` | 对话 API 路由 |
| `backend/routers/coach_results.py` | 教练结果 API |
