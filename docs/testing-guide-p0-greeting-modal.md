## 场景 1：无画像用户（no_profile stage）

**准备**: 新注册账号，从未上传简历，profile 无任何内容

**验证**:
1. 打开 Chat 面板
2. ✅ Greeting 应为：
   > 嗨！我是你的职业成长教练。
   > 我们先从了解你开始——上传一份简历，我帮你做能力画像和方向分析。
3. ✅ Chips：「这个系统能做什么？」「我是计算机专业学生」「前端和后端怎么选」
4. ✅ **不应出现**「画像建好了」等有画像才有的文案

**API 验证**（DevTools → Network）:
```
GET /api/chat/greeting
→ { "stage": "no_profile", "greeting": "嗨！..." }
```

---

## 场景 2：有画像 · A 类用户（has_profile, no goal, job_target 为空）

**准备**: 已上传简历，skills > 0，简历无明确求职意向字段，未设目标

**验证**:
1. ✅ Greeting 三段结构：
   - 第一段：「画像建好了！我识别到你有 X 项技能，XXX。」
   - 第二段：「现在不用急着定目标——先搞清楚哪个方向真的适合你，比随便选一个再后悔要好。」
   - 第三段：「你有感兴趣的方向吗？或者让我帮你分析最匹配的。」
2. ✅ Chips：「帮我分析最匹配的方向」「我对某个方向感兴趣」「我还不确定方向怎么办」
3. ✅ **不应出现**「去画像页查看推荐」「诊断一份JD」

---

## 场景 3：有画像 · B 类用户（has_profile, no goal, job_target 非空）

**准备**: 已上传含「求职意向: 前端开发工程师」的简历，未设目标

**验证**:
1. ✅ Greeting：
   > 画像建好了，XXX！我看到你的求职意向是「前端开发工程师」。
   > 先别急着定目标——让我帮你分析这个方向的真实市场情况和你的差距，看清楚再做决定。
2. ✅ Chips 第一项：「分析「前端开发工程师」方向」（动态填入 job_target）

---

## 场景 4：有画像 · 已设目标（has_profile + goal）

**准备**: 已上传简历，已设定职业目标

**验证**:
1. ✅ Greeting 提及目标名称，引导做 JD 诊断
2. ✅ Chips：「诊断一份JD」「成为 X 核心要什么」「我现在差距有多大」
3. ✅ **不应出现**「去画像页」等跳转引导

---

## 场景 5：目标确认弹窗（核心路径）

**准备**: 有真实画像的账号，访问任意岗位详情页（如 /roles/backend-engineer）

**步骤**:
1. 滚动到页面底部，点击「开始探索「XXX」方向」
2. ✅ 弹窗出现，**不立即设定目标**
3. ✅ 弹窗包含：
   - 标题：「确认设定职业方向」
   - 蓝色方向 badge（显示正确岗位名）
   - 三条说明（JD诊断 / 技能差距追踪 / 成长档案）
   - 安心说明：「方向可以随时更换，不会丢失...」
4. 点「再想想」→ ✅ 弹窗关闭，目标未设定
5. 点弹窗背景遮罩 → ✅ 同样关闭
6. 重新打开弹窗，点「确认，开始追踪」
   - ✅ 按钮变「设定中...」（disabled 状态）
   - ✅ 成功后跳转到 /growth-log
   - ✅ growth-log 显示刚才设的目标方向

## 场景 6：无画像用户访问岗位详情页

2. ✅ 点击跳转到 /profile?from=goal-set
