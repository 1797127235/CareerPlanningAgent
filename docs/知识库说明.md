# 本地知识库资料说明

> 第十五届中国大学生服务外包创新创业大赛 · A 类 · 陕西明杉数据科技
> 作品名称：职途智析——基于AI的大学生职业规划智能体

---

## 知识库总览

本系统共部署 **6 大类本地知识库**，总数据量约 **500 MB**，涵盖岗位画像、AI 影响评估、面试知识体系、技能学习路径、市场趋势信号、教练对话规则。所有数据均存储于项目本地，不依赖外部 API 调用，确保系统在无网络环境下仍可完成核心诊断与推荐功能。

| 知识库类别 | 文件/目录 | 规模 | 核心用途 |
|-----------|----------|------|---------|
| 岗位知识图谱 | `data/graph.json` | ~45 节点 / 101 条边 | 岗位画像、晋升路径、换岗路线 |
| AEI 权威数据 | `data/EconomicIndex/` | **409 MB** | AI 替代压力评估、协同杠杆计算 |
| 面试方向知识库 | `backend/interview_skills/_shared/references/` | 8 个方向文档 | 模拟面试题目生成、知识覆盖校验 |
| 技能补法路径 | `data/skill_fill_path_tags.json` | ~200 技能标签 | 技能缺口分类（学/做/先学后做） |
| 市场信号 | `data/market_signals.json` | 多维度趋势数据 | 行业趋势、薪资中位数参考 |
| Coach Skill 规则 | `agent/skills/coach-*/SKILL.md` | 13 个场景 | AI 教练对话策略、意图分类规则 |

## 1. 岗位知识图谱

### 文件
- `data/graph.json` — 主图谱数据（45 节点、101 条转换边）
- `data/岗位数据.csv` — 命题企业提供原始数据（约 10,000 条）
- `data/节点数据.csv`、`data/节点边.csv` — 图谱构建中间产物

### 内容说明

| 字段 | 说明 |
|------|------|
| **岗位节点** | 45 个，覆盖 20 个岗位族（Java 后端 / 前端 / C++ / 算法 / 产品 / 测试等） |
| **晋升路径** | 每节点 5 级：初级 → 中级 → 高级 → 专家 → 架构师/总监 |
| **技能分层** | core / important / bonus 三级，每项带出现频率 |
| **软技能画像** | 沟通、学习、抗压、创新、协作五维度评分 |
| **项目推荐** | 每节点 2-3 个典型实战项目 + 推荐理由 |
| **AI 影响叙事** | 基于 AEI 数据的冲击与机遇分析（文字描述） |
| **市场指标** | 替代压力（0-100）、薪资中位数、人机协作杠杆 |
| **换岗路径** | 101 条转换边，四因子成本模型（技能重叠度、经验迁移成本、薪资落差、学习周期） |

### 数据来源
- 智联招聘数据库（2016-2025，~100 万条）
- 上市公司招聘大数据（2014-2026，~50 万条）
- 企业命题提供岗位数据（9,959 条）

## 2. AEI 权威数据（Anthropic Economic Index）

### 文件
- `data/EconomicIndex/` — 完整数据集（**409 MB**）

### 内容说明

Anthropic 官方在 HuggingFace 公开的权威数据集，基于**数百万条真实 Claude 对话**聚合而成，精确刻画 AI 在现实经济中的任务级渗透和职业级暴露。

**包含 4 个时间版本**：
- `release_2025_02_10`
- `release_2025_03_27`
- `release_2025_09_15`
- `release_2026_01_15`

**六维分解**（自动化 vs 增强）：

| 维度 | 含义 | 方向 |
|------|------|------|
| directive | 指令式自动化 — AI 全权执行 | 替代 |
| feedback_loop | 反馈循环自动化 — AI 执行、人类校验 | 替代 |
| task_iteration | 迭代增强 — 人类主导、AI 协助 | 增强 |
| validation | 验证辅助 — AI 帮人类检查 | 增强 |
| learning | 学习辅助 — AI 帮人类学习 | 增强 |
| filtered | 无法分类 | — |

**系统核心评分**：
- `replacement_pressure`（AI 替代压力 0-100）
- `human_ai_leverage`（人+AI 协同杠杆 0-100）
- `zone`（战略定位：leverage / transition / caution / critical）

### 语义对齐方法
通过 `text-embedding-v3` 完成中文岗位任务 → O*NET 英文任务的语义对齐，让每个岗位的"AI 替代压力"都有权威数字支撑。

## 3. 面试方向知识库

### 文件
`backend/interview_skills/_shared/references/*.md`

| 文件 | 方向 | 内容 |
|------|------|------|
| `cpp-core.md` | C++ 系统开发 | C++ 语言核心、操作系统、网络编程、多线程并发、系统设计 |
| `frontend-core.md` | 前端开发 | JavaScript/TypeScript、React/Vue、浏览器原理、性能优化、工程化 |
| `java-core.md` | Java 后端 | JVM、Spring 生态、数据库、分布式、微服务 |
| `algorithm.md` | 算法工程师 | 数据结构、机器学习、深度学习、NLP/CV、算法设计 |
| `product.md` | 产品经理 | 需求分析、用户研究、数据驱动、项目管理、商业化 |
| `test-dev.md` | 测试开发 | 自动化测试、性能测试、CI/CD、测试框架、质量保障 |
| `os-network.md` | 系统/网络（共享） | 操作系统原理、计算机网络、Linux 内核 |
| `system-design.md` | 系统设计（共享） | 分布式系统、高并发架构、缓存、消息队列 |

### 用途
模拟面试题目不是死题库，而是基于知识库动态生成。LLM 根据用户画像（技能水平、项目经历、gap 技能）从知识库中选取考点，生成个性化面试题。

## 4. 技能补法路径分类

### 文件
- `data/skill_fill_path_tags.json`

### 内容说明

将技能缺口按最佳学习路径分为三类：

| 类别 | 典型技能 | 建议路径 |
|------|---------|---------|
| 📚 learn（系统学习） | 数据结构、操作系统、算法 | 在线课程 / 教材 / 刷题 |
| 🛠️ practice（项目实践） | 高并发、分布式、性能优化 | 做项目才能真正掌握 |
| 🔀 both（先学后做） | Docker、K8s、Git | 先学概念，再做项目巩固 |

约 **200 个技能**已标注分类标签，用于发展报告中的"补法地图"生成。

## 5. 市场信号

### 文件
- `data/market_signals.json`
- `data/industry_signals.json`
- `data/skill_frequencies.json`

### 内容说明

| 数据 | 说明 |
|------|------|
| 行业趋势 | IT 各细分方向的热度变化、人才供需比 |
| 技能频率 | 各技能在招聘 JD 中的出现频率（用于匹配度计算权重） |
| 薪资中位数 | 按岗位/城市/经验年限分层的薪资参考 |

用于 AI 教练对话中的"市场信号"回复（如"Java 后端近半年需求增长 12%"）。

## 6. Coach Skill 对话规则

### 文件
`agent/skills/coach-<scenario>/SKILL.md`（共 13 个）

| Skill | 场景 | 用途 |
|-------|------|------|
| `coach-greeting` | 开场问候 | 首次对话建立信任 |
| `coach-confirmation` | 确认理解 | 复述用户意图，确认无误 |
| `coach-concern-direct` | 直接关切 | 用户表达焦虑时的回应 |
| `coach-emotional-support` | 情感支持 | 挫败感、迷茫时的共情 |
| `coach-comparison-detox` | 比较解毒 | 用户与他人比较时的引导 |
| `coach-direction-scaffold` | 方向搭建 | 帮助用户梳理候选方向 |
| `coach-progress-report` | 进度汇报 | 查看成长档案总结 |
| `coach-interview-prep` | 面试准备 | 面试技巧、常见问题 |
| `coach-resume-review` | 简历诊断 | 简历优化建议 |
| `coach-market-signal` | 市场信号 | 行业趋势、岗位需求 |
| `coach-request-deliver` | 交付请求 | 用户要求生成报告/计划 |
| `coach-project-planning` | 项目规划 | 帮助学生规划项目 |
| `coach-decision-socratic` | 决策苏格拉底 | 用提问引导用户自己做决定 |

### 架构特点
采用 **Progressive Disclosure 模式**：
- SystemMessage 只注入 Skill Catalog（13 个 name + description，~830 tokens）
- LLM 判断场景后调 `load_skill(name)` tool 按需加载完整规则
- 相比全量 push 节省 ≥ 60% token，扩到 20 个 Skill 也只吃 ~800 tokens base cost

## 数据安全与隐私

| 措施 | 说明 |
|------|------|
| 本地存储 | 所有用户数据（简历、画像、诊断记录）存储于本地 SQLite |
| 不上传原始简历 | 简历解析后只保留结构化数据，原始文件不传输到第三方 |
| 等保三级规范 | 遵循教育行业标准与等保三级规范 |
