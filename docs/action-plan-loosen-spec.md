# action-plan SKILL 松绑 spec（交给 Kimi 执行）

## 目的

当前 `backend/skills/action-plan/SKILL.md` 的输出读起来像打卡模板：

- 固定三阶段（0-2 周 / 2-6 周 / 6-12 周），信号稀也要硬凑
- 阶段标题固定成"立即整理 / 技能补强 / 项目冲刺+求职"
- 每条 item 强拆 observation（30-130 字）+ action（30-80 字）两栏
- LLM 被压在框里只能套"观察 X → 行动 Y"的句式

给 LLM 更大发挥空间，同时**保留防假大空的锚**（引具体证据 / 动作带时间和产出 / 禁词 / 字段名不外泄）。

---

## 仅改一个文件

**`backend/skills/action-plan/SKILL.md`**

其它文件**不要动**：
- `backend/services/report/pipeline.py::_coerce_action_plan` 已做 sanitizer 兜底，保持不变
- `frontend/src/components/report/ChapterIV.tsx` 已支持 `observation + action` 两字段分别渲染，保持不变
- `_coerce_action_plan` 要求 `stages[].items[]` 里每条有 `observation` 和 `action` 字段（前端按钮 prefill 用 `action || text`），**这两个字段不能废**

## 要改的部分

### 改动 1：`### 最高原则：反套话` 这节重写

**原文**（要整节替换）：

```
### 最高原则：反套话

**每条 observation 必须至少满足以下之一：**
- 引用 `profile_core.work_experience` 里某段**具体的职位/公司/年限/职责描述**
- 引用 `profile_core.projects` 里某个**具体的项目名或技术栈描述**
- 引用 `profile_core.education` 的**具体学历/专业**
- 引用 `profile_core.personal_statement` 里的**原话**（≤15 字摘录）
- 引用 `growth_entries` 里某条**具体记录**（面试公司/轮次、学习笔记原文、项目进展、未完成计划）
- 引用 `milestones` 里某个**具体事件**

**如果这六个来源都没有足够信息支撑某条建议——直接删掉这条，不要硬写。**
```

**改写成**：

```
### 最高原则：像导师说话，而不是像表格生成器

你面对的是一个具体的学生，不是一份 JSON。你要做的事：**读完他的材料，以一位真正懂他处境的导师的口吻，告诉他接下来该做什么**。

想象你就坐在他对面，看过他的简历、他的成长档案、他最近都在忙什么。你开口说的每一句话，都必须是**只对他有意义**的——把同样的话换一个学生看，他会觉得不适用。

这意味着：

- **引用具体**——项目叫什么名字、哪家公司哪个岗位、哪门课、哪个技术栈、哪条成长档案里记了什么、哪次面试卡在哪。至少有一处是他独有的锚点。
- **承接事实，不推测**——基于"他做过/没做/正在做什么"说话，不写"可能/或许/似乎"。
- **删胜于凑**——如果某个建议找不到他身上的具体依据，直接不写这条。宁可整份计划只有两件事，也不要凑一堆万能建议。

可引用的锚点（这些都是 `summary_json` 里 LLM 能看到的字段，仅供你内部参考 —— **正文里绝不能出现这些字段名本身**，必须翻译成自然中文）：

- 工作经历：公司 / 岗位 / 年限 / 职责描述
- 项目：项目名 / 技术栈 / 描述里的具体技术点
- 教育：学校 / 专业 / 学历
- 个人陈述原话（摘录 ≤15 字）
- 成长档案的具体记录：面试公司与轮次、学习笔记、项目进展、未完成计划
- 里程碑事件

字段名到自然中文的翻译办法见下面《字段名禁止外泄》一节。
```

### 改动 2：`### 硬约束` 这节重写

**原文**（要整节替换）：

```
### 硬约束

1. **不重复 completed_since_last_report**：学生已完成 X，不可再建议"学 X"
2. **不绑定具体项目名**："在 XX 项目基础上加 Y" 禁用；可以说"你的 XX 类项目"
3. **每条 item 必须同时包含 observation 和 action**
   - **observation**：30-130 字，陈述"基于你做过什么 / 留下了什么证据 / 眼下缺的是哪块"，禁祈使句、禁推测性副词（"可能/或许/似乎"）
   - **action**：30-80 字，祈使句，**必须有**：动作动词 + 具体范围 + 时间或产出
     正例："3 天内整理 RDB vs AOF 对比图，做 2 道相关面试题"
     反例："加强 Redis 学习"（无具体动作）/ "掌握 Redis 持久化"（无时间/产出）
4. **信号稀缺时允许稀疏输出**——如果 `milestones + growth_entries + profile_core.work_experience + profile_core.projects` 加起来证据不足以支撑三阶段，**只输出 1-2 个阶段，每个阶段 1-2 条**。不许为了凑数把"刷 LeetCode / 写 HTTP 服务器 / 模拟面试"作为通用建议塞进去。
5. **信号充足时标准结构**：
   - stage 1（0-2 周，求职准备类，1-2 条）
   - stage 2（2-6 周，技能补强类，1-3 条）
   - stage 3（6-12 周，项目冲刺/求职推进，1-2 条）
6. **observation 与 action 互补**，不重复。observation 描述"为什么要做"，action 描述"具体怎么做"
7. **输出严格 JSON**，不要 markdown 代码块
```

**改写成**：

````
### 硬约束

#### 关于阶段（stages）

1. **阶段数量你自己决定**：1 到 4 个之间。看这个学生眼下的信号密度和处境：
   - 信号稀、画像薄、成长档案空 → 1 个阶段，1-2 条事情，把真能落地的先写完
   - 信号密、项目多、面试在进行中 → 2-3 个阶段，按他自己的节奏分
   - 只有在真的需要长线（3 个月以上）叙事时才用 4 个阶段
2. **阶段时长你自己决定**：不强制 0-2 周 / 2-6 周 / 6-12 周。可以写"这两周"、"接下来一个月"、"面试季前"、"毕业答辩之后"——贴着他的时间锚走。
3. **阶段标题要具体**：别写"立即整理 / 技能补强 / 项目冲刺"这类万能词。要写出这个阶段到底要干的那件事——比如"把网络库的性能数据补全"、"拿下第一个 offer 之前把简历收紧"、"边投递边把两个最大的技能空白补上"。
4. **每个阶段一句里程碑**：阶段结束时，学生手里具体多出什么东西（一份文档 / 一次完整面试 / 一个可跑的 demo / N 家投递）。

#### 关于条目（items）

5. **不重复已完成的事**（`completed_since_last_report` 里列的）——学生已经做完 X，不要再建议"学 X"。
6. **不硬点具体项目名当动作**："在 [项目 A] 上加 [技术 B]" 这种绑定禁用。可以说"你简历里那个偏 [方向] 的项目"。
7. **每条 item 必须有 `observation` 和 `action` 两个字段**，但不要当表格填：
   - `observation`：**用自然语气写**，一段通顺的话，不限字数（大致 2-4 句就够）。说清楚"你做过什么 / 现在卡在哪 / 为什么这事值得花时间"。**禁用祈使句、推测性副词**。
   - `action`：**一句话**祈使句，必须包含：**动作动词 + 具体范围 + 时间锚或交付物**。前端会把这句话作为"记到成长档案"的 prefill，所以要自成一句、可直接贴进档案。
     - 正例："这周内把 [技术 A] vs [技术 B] 的对比整理成一张图，顺带刷 2 道相关面试题。"
     - 正例："月底前完成一次模拟面试录像，针对 [方向] 回答至少 5 个追问。"
     - 反例："加强 [技术] 学习。"（没动作、没时间）
     - 反例："掌握 [技术] 的核心原理。"（没交付物）
8. **observation 和 action 互补不重复**：observation 是为什么 + 背景，action 是具体怎么做 + 何时做出什么。
9. **信号不足就少写**：如果某个方向没有具体材料支撑，就不写那条；如果整体材料稀，就只出 1 个阶段 1-2 条，不要塞"刷 LeetCode / 写 HTTP 服务器 / 模拟面试"这种人人都能套的填充题。

#### 关于类型（item.type）

10. `type` 取 `skill` / `project` / `job_prep` 之一。选哪个看这件事**本质上**属于哪类，不受"哪个阶段该放哪类"的束缚——技能事可以出现在任何阶段，求职事也一样。

#### 关于输出

11. **严格 JSON**，不要 markdown 代码块，不要解释文字。
````
